## Минимальный набор сущностей, которые нужно создать в системе

### 1. Commerce / Party

| Сущность | Что создаётся |
|---|---|
| `party` | карточка контрагента хозяйства |
| `jurisdiction` | `RU`, если в компании ещё не подтянута автоматически |
| `party relation` | не обязательна для первого теста |

### 2. CRM

| Сущность | Что создаётся |
|---|---|
| `account` | хозяйство как рабочий контур для `TechMap` |
| `contact` | минимум один контакт под внешнего гл. агронома хозяйства |

### 3. Field / Season / Planning

| Сущность | Что создаётся |
|---|---|
| `field` | `KZM-01` |
| `season` | `2026/2027` |
| `cropZone` | под `RAPESEED` |
| `harvestPlan` | черновой план под генерацию техкарты |

### 4. Front-office

| Сущность | Что создаётся |
|---|---|
| `invitation` | приглашение внешнему гл. агроному хозяйства |
| `counterpartyUserBinding` | привязка внешнего пользователя к хозяйству |

## Минимум данных, без которых генерация техкарты не имеет смысла

| Блок | Минимум |
|---|---|
| Хозяйство | название, адрес, рабочий `account` |
| Поле | имя, площадь, контур, привязка к хозяйству |
| Сезон | год/сезон, привязка к полю |
| Культура | `озимый рапс` |
| Агроконтекст | предшественник, тип почвы, базовая агрохимия |
| Внешний пользователь | приглашённый гл. агроном хозяйства |

## Регистрационный комментарий для команды

- В этом тесте мы не пытаемся сразу завести все поля хозяйства.
- В систему заводится один целевой массив `KZM-01`, достаточный для первого `end-to-end` прогона.
- Юрданные и `ЛПР` не выдумываются и должны быть введены пользователем вручную.
