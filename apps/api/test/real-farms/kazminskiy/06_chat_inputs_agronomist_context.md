# Готовые тексты от внешнего гл. агронома хозяйства для `clarification loop`

## Как использовать

Этот файл нужен для ручного теста сценария, где внешний гл. агроном хозяйства пишет в `front-office`, а система и внутренний контур поэтапно добирают недостающие данные.

Это **не анкета целиком**. Это пакет стартового сообщения и готовых ответов на наиболее вероятные gaps.

## 1. Первичное сообщение от хозяйства

```text
Нужно подготовить техкарту по озимому рапсу на сезон 2026/2027.

Хозяйство: СПК колхоз племзавод «Казьминский».
Поле: KZM-01.
Площадь поля: 1006 га.
Локация: район села Казьминское, Ставропольский край.

Планируем озимый рапс после озимой пшеницы.
Поле большое, неоднородное, внутри есть более влажные и более сухие части.
Хотим получить рабочую техкарту с учётом местных условий, техники и ограничений по срокам.
Целимся в урожайность около 3.1 т/га, без завышенных ожиданий.
```

## 2. Если система спросит про предшественник

```text
Предшественник — озимая пшеница.
Уборка предшественника обычно завершается в конце июня или начале июля.
После уборки остаётся значимый объём пожнивных остатков, распределение соломы не всегда идеально ровное.
```

## 3. Если система спросит про почву

```text
Почва по полю в основном чернозёмная, тяжёлый и средний суглинок.
По массиву есть неравномерность.

pH примерно от 6.5 до 7.1.
Фосфор средний, ориентир 16–24 мг/кг.
Калий достаточный, ориентир 280–340 мг/кг.
Сера слабое место, ориентир 6–9 мг/кг.
Бор местами на нижней границе, ориентир 0.18–0.28 мг/кг.
Есть локальное уплотнение на глубине 20–35 см.
```

## 4. Если система спросит про неоднородность поля

```text
Поле формально одно, но внутри три выраженные зоны.

Северо-западная часть лучше держит влагу.
Центральная часть самая типичная по массиву.
Юго-восточная часть суше и быстрее теряет стартовую влагу в конце августа.
```

## 5. Если система спросит про технику

```text
По тяге хозяйство закрыто нормально.
Есть несколько мощных тракторов, два основных посевных агрегата, один основной опрыскиватель и несколько комбайнов.

Главное ограничение — один основной опрыскиватель и неидеальный резерв по посевному окну на массив 1006 га.
По уборке не весь контур одинаково хорошо оснащён именно под рапс.
```

## 6. Если система спросит про сроки посева

```text
Базово хотим войти в посевное окно с 20 августа по 5 сентября.
Но на таком массиве понимаем риск растяжки срока до 8–10 сентября, если погода или техника задержат работы.
```

## 7. Если система спросит про целевую урожайность

```text
Консервативный минимум считаем 2.4 т/га.
Рабочая цель — 3.1 т/га.
Хороший результат при сильном исполнении — 3.4 т/га.
Максимум выше этого считаем уже агрессивным сценарием.
```

## 8. Если система спросит про ключевые риски хозяйства

```text
Основные риски для нас:

1. неравномерная стартовая влага по полю;
2. риск растянуть посев на слишком длинное окно;
3. слабое место по сере и бору;
4. зависимость части операций от одного опрыскивателя;
5. разная сила розетки по зонам массива перед зимой.
```

## 9. Если система спросит про пожелания к техкарте

```text
Нужна не идеальная теоретическая карта, а рабочая управленческая техкарта.
Нужно, чтобы карта учитывала:

- масштаб массива;
- неоднородность поля;
- реальные сроки;
- технику хозяйства;
- критичные контрольные точки по осенней розетке и перезимовке.
```

## 10. Если система запросит недостающие данные дополнительно

Короткое правило для ручного теста:

- отвечать кратко;
- давать только запрошенный gap;
- не пытаться сразу переслать всю “анкету”;
- сохранять один и тот же контур:
  - поле `KZM-01`
  - площадь `1006 га`
  - сезон `2026/2027`
  - культура `озимый рапс`
  - цель `3.1 т/га`

## 11. Что считается хорошим результатом диалога

Хороший результат — это когда после нескольких сообщений система приходит к состоянию:

- контекст поля понятен;
- ключевые gaps закрыты;
- генерация техкарты выглядит обоснованной;
- explainability показывает, почему ветка именно озимая и почему обязательны контрольные точки по осени и перезимовке.
