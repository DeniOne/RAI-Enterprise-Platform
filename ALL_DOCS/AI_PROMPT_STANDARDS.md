# AI Prompt Standards (Стандарты промптов) 📝

> **Статус:** Инженерный стандарт | **Версия:** 1.0 | **Слой:** Engineering / Guidelines

---

## 1. Архитектурные требования
Все промпты в системе должны следовать принципу **Stateless Architecture**:
- Запрещено полагаться на внутреннюю память модели (OpenAI threads).
- Весь необходимый контекст должен быть упакован в один запрос.

---

## 2. Структура Промпта (The Canon Grid)

Каждый промпт должен состоять из четырех блоков:

1. **Role & Identity**: Определение роли (напр. "Ты — ведущий агроном-эксперт по рапсу").
2. **Context Injection**: 
   - Данные из Business Core (фильтрованные по `companyId`).
   - Релевантные Энграммы (Positive/Negative).
3. **Task & Constraints**: Четкое задание и список ограничений (напр. "Не предлагай препараты, которых нет на складе").
4. **Output Format**: Форматирование ответа (JSON для системных действий, Markdown для человека).

---

## 3. Безопасность и Multi-tenancy

> [!IMPORTANT]
> **Data Leaks Prevention**
> Промпт-инженер обязан гарантировать, что данные Компании А никогда не попадут в запрос, инициированный Компанией Б.

**Правила**:
- Использовать `TenantContextWrapper` для автоматической фильтрации данных перед сборкой промпта.
- Запрещено передавать в LLM: пароли, личные телефоны, адреса проживания сотрудников.

---

## 4. Работа с Галлюцинациями
Для минимизации ошибок:
- Использовать **Low Temperature** (0.2 - 0.3) для технических расчетов.
- Добавлять инструкцию: "Если в контексте недостаточно данных для ответа — скажи 'Данных недостаточно' и не выдумывай".

---

## 5. Версионирование
Все "боевые" промпты должны храниться как отдельные файлы/шаблоны в `apps/api/src/modules/ai-orchestration/prompts/` и иметь версию. Изменение промпта приравнивается к изменению кода и требует тестов.
