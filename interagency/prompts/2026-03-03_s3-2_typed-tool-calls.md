# PROMPT — 3.2 Typed Tool Calls only (LAW)
Дата: 2026-03-03  
Статус: active  
Приоритет: P0 (Architectural Law)  

## Цель
Цементировать «Закон типизированных вызовов» (Typed Tool Calls only). Агент (LLM) обязан взаимодействовать с доменной логикой, данными и внешними системами ИСКЛЮЧИТЕЛЬНО через реестр инструментов (`RaiToolsRegistry`). Любые формы динамического исполнения строк (string-execution), `eval`, `new Function` или нетипизированные вызовы запрещены.

## Контекст (Truth Sync)
- Реестр: `apps/api/src/modules/rai-chat/tools/rai-tools.registry.ts`.
- Типы: `apps/api/src/modules/rai-chat/tools/rai-tools.types.ts`.
- Аудит сейчас: логируется только факт вызова, статус и причина ошибки. Пэйлоады не сохраняются.

## Задачи

### 1. Hardening Registry Audit (Forensic Mode)
- Обновить `RaiToolsRegistry.logToolCall`:
  - Добавить пэйлоад вызова в лог (сериализованный JSON).
  - Убедиться, что лог содержит `traceId` и `companyId` (уже есть, не сломать).
  - Пэйлоад должен логироваться как в случае успеха, так и в случае ошибки валидации/исполнения.

### 2. Enforcement (Law Compliance)
- Провести аудит `apps/api/src/modules/rai-chat/rai-chat.service.ts` и `RaiToolsRegistry`:
  - Исключить/запретить любые методы, позволяющие исполнить логику в обход `execute` с Joi-валидацией.
  - Убедиться, что `execute` — единственный публичный метод шлюза инструментов.

### 3. Верификация & Тесты
- Обновить `rai-tools.registry.spec.ts`:
  - Добавить тест, проверяющий, что при вызове инструмента в `Logger.log` или `Logger.warn` попадает полный JSON с пэйлоадом.
  - Проверить сценарий с невалидным пэйлоадом — лог должен содержать ошибочные данные для Forensic-анализа.
- Выполнить `tsc` для `apps/api`.

## Definition of Done
- [ ] `RaiToolsRegistry` логирует пэйлоады всех вызовов (успех/провал).
- [ ] К инструментам нельзя обратиться в обход типизированного шлюза.
- [ ] Тесты в `rai-tools.registry.spec.ts` подтверждают Forensic-логирование пэйлоадов.
- [ ] `apps/api` проходит `tsc`.
- [ ] Подготовлен отчёт по канону Software Factory.

## Запреты
- НИКАКИХ правок в самих инструментах (бизнес-логика `echo` или `snapshot` не меняется).
- НИКАКИХ изменений в `workspaceContext`.
- НИКАКИХ `any` в сигнатурах методов реестра.
