# PROMPT — 4.1 Chat Widget Logic (Domain Bridge)
Дата: 2026-03-03  
Статус: active  
Приоритет: P1 (Feature Bridge)  

## Цель
Перевести формирование виджетов чата со статических заглушек на динамическую доменную логику. Агент должен возвращать виджеты, которые отражают реальное состояние системы (отклонения, задачи) для конкретной компании (`companyId`) и рабочего контекста (`workspaceContext`).

## Контекст (Truth Sync)
- Рендерер: `apps/web/components/ai-chat/AiChatWidgetsRail.tsx`.
- Схемы: `apps/api/src/modules/rai-chat/widgets/rai-chat-widgets.types.ts`.
- Текущая логика: `RaiChatService.buildWidgets` (hardcoded).

## Задачи

### 1. Refactoring buildWidgets
- Вынести логику формирования виджетов из `RaiChatService` в отдельную функцию или компактный вспомогательный сервис (например, `RaiChatWidgetBuilder`).
- Логика должна принимать:
  - `companyId: string`
  - `workspaceContext: WorkspaceContextDto`
- **Заглушки v2**: Оставить заглушки только для тех доменных областей, где API еще не реализовано, но сделать их зависимыми от контекста (например, подставлять ID компании или название маршрута из контекста в текст виджета).

### 2. Domain Data Link (Mock Data Separation)
- Подготовить архитектурное место для внедрения реальных репозиториев/сервисов (например, `TasksService` или `DeviationsService`), когда они появятся.
- Для `DeviationListWidget`: если в контексте есть `route`, генерировать "обнаруженные" отклонения именно для этого маршрута.
- Для `TaskBacklogWidget`: показывать задачи, релевантные текущему пользователю/компании.

### 3. Forensic & Quality
- Убедиться, что возвращаемые виджеты проходят валидацию в `AiChatResponseDto`.
- Проверить, что `Forensic-логирование` (задача 3.2) корректно сохраняет эти виджеты в составе ответа.
- Написать тесты на `WidgetBuilder` (или обновленный `RaiChatService`), подтверждающие динамическое изменение контента виджетов при смене `workspaceContext`.

## Definition of Done
- [ ] `RaiChatService` возвращает виджеты, наполненные динамически на основе контекста.
- [ ] Логика формирования виджетов изолирована от основного сервиса чата.
- [ ] Тесты подтверждают реакцию виджетов на `route` и `companyId`.
- [ ] `apps/api` проходит `tsc`.

## Ограничения
- НИКАКИХ правок в UI компонентах фронтенда (они уже умеют это рендерить).
- НИКАКИХ правок в базовых типах виджетов (использовать текущие `DeviationList` и `TaskBacklog`).
- НИКАКИХ `any`.
