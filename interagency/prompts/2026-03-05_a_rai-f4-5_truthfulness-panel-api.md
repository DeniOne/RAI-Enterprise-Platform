# PROMPT — Truthfulness Panel API (Phase 4.5)
Дата: 2026-03-05
Статус: active
Приоритет: P0

## Цель
Поднять слой API для дашборда Truthfulness/Quality Panel. Этот эндпоинт должен агрегировать данные из `TraceSummary` и отдавать статистику: средний BS%, Evidence Coverage, и список худших трейсов (worst traces) для конкретного тенанта (`companyId`) за указанный период (окно).

## Контекст
- **Почему это важно сейчас:** Мы научились считать BS% (Фаза 4.4) и собирать трейсы с таймлайнами (Фаза 4.1-4.2). Теперь нужно выдать эти метрики на фронтенд (в дашборд), чтобы супервизоры могли видеть радар честности своих агентов.
- **Связанные документы:**
  - `docs/00_STRATEGY/STAGE 2/A_RAI_IMPLEMENTATION_CHECKLIST.md` (Phase 4, пункт 5).

## Ограничения (жёстко)
- **Изоляция (Tenant Isolation):** Агрегация СТРОГО с фильтром по `companyId` контекста.
- **Производительность:** Считать агрегаты на лету для тысяч трейсов дорого. В идеале нужно делать выборку (например, за последние 24-72 часа) или использовать встроенные функции агрегации БД (Prisma `.aggregate` / `.groupBy`).
- **Сайд-эффекты:** Строго READ-ONLY.
