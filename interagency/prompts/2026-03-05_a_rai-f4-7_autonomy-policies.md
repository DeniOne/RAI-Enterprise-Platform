# PROMPT — Autonomy Policies by BS% (Phase 4.7)
Дата: 2026-03-05
Статус: active
Приоритет: P0

## Цель
Реализовать механизм применения политик автономности агентов на основе ранее вычисляемой метрики `bsScorePct` (уровня галлюцинаций/ошибок). Система должна автоматически понижать уровень автономности, если средний BS% превышает допустимые пороги (например, при BS% > 30% переводить в режим `QUARANTINE` или `TOOL_FIRST` с обязательным аппрувом человека).

## Контекст
- **Почему это важно сейчас:** Мы научились считать BS% (Фаза 4.4) и алертить об отклонениях (Фаза 4.6), а также выводить дашборды (Фаза 4.5). Теперь нужно замкнуть цикл: система должна **автоматически реагировать** на плохие метрики. Агент, который начинает ловить галлюцинации, теряет право автономно менять Техкарты или вызывать критичные тулзы.
- **Связанные документы:**
  - `docs/00_STRATEGY/STAGE 2/A_RAI_IMPLEMENTATION_CHECKLIST.md` (Phase 4, пункт 4.4 "Управление Автономностью").

## Ограничения (жёстко)
- **Изоляция:** Правила автономности рассчитываются и применяются строго в рамках одного `companyId`.
- **Интеграция:** У нас уже есть `Two-Person Rule` (RiskPolicies / Phase 3). Политика автономности должна органично встраиваться в этот слой: если агент в `QUARANTINE`, его вызовы опасных (mutating) тулзов блокируются или принудительно эскалируются на `SupervisorAgent`/человека.
- **Динамичность:** Статус автономности вычисляется динамически на базе окна метрик (например, последние 10-20 `TraceSummary`).

## Задачи (что сделать)
- [ ] Описать enum `AutonomyLevel` с уровнями (например: `AUTONOMOUS`, `TOOL_FIRST`, `QUARANTINE`).
- [ ] Реализовать сервис `AutonomyPolicyService` со методом `getCompanyAutonomyLevel(companyId: string)`.
  - Логика: берём средний `bsScorePct` из `TraceSummary` за окно X часов.
  - Если `< 5%` → `AUTONOMOUS`.
  - Если `[5% .. 30%]` → `TOOL_FIRST` (агент теряет право на auto-execution опасных тулзов, все экшены идут через подтверждение).
  - Если `> 30%` → `QUARANTINE` (агент может только читать данные, никаких мутаций).
- [ ] Интегрировать эту проверку в `SupervisorAgent.orchestrate()` (или `ToolExecutor` / `RiskPolicyEngine`). Если тулз опасный, а уровень `QUARANTINE` → `ForbiddenException` или `RequireEscalation`.
- [ ] Покрыть сервис и логику деградации автономности unit-тестами.

## Definition of Done (DoD)
- [ ] `AutonomyLevel` вычисляется корректно.
- [ ] Интеграция в пайплайн тулзов реализована (опасные тулзы блокируются/эскалируются при высоком BS%).
- [ ] Тесты PASS (`tsc`, `jest`).

## Тест-план (минимум)
- [ ] Unit: BS% = 2% -> метод возвращает `AUTONOMOUS`, выполнение тулзов идёт штатно.
- [ ] Unit: BS% = 15% -> метод возвращает `TOOL_FIRST`.
- [ ] Unit: BS% = 40% -> метод возвращает `QUARANTINE`, вызов опасного тулза (например, changeTechMap) выбрасывает ошибку/эскалацию.
