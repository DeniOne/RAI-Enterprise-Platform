# PROMPT — Evidence Tagging MVP (Phase 4.3)
Дата: 2026-03-05
Статус: active
Приоритет: P0

## Цель
Реализовать MVP механизма `Evidence Tagging`. Агенты должны привязывать свои утверждения (claims) к конкретным источникам данных (результат тулза, строка БД, кусок RAG-документа). Это фундамент для дальнейшего расчета честности (BS%).

## Контекст
- **Почему это важно сейчас:** Мы не можем считать BS% (уровень пиздежа), если не знаем, на что опирался LLM. Нам нужен механизм, который заставит агента вместе с ответом возвращать ссылки на Evidence.
- **Связанные документы:**
  - `docs/00_STRATEGY/STAGE 2/A_RAI_IMPLEMENTATION_CHECKLIST.md` (Phase 4, пункт 3).

## Ограничения (жёстко)
- **Структура ответа LLM:** Строго через Structured Output (JSON Schema) или Tool Use, чтобы LLM физически не могла ответить текстом без массива `evidenceRefs`.
- **Backward Compatibility:** Если агент (например, тупой stub) не может вернуть evidence, он возвращает пустой массив. Система не должна падать.
- **Tenant Isolation:** Источники (Evidence) всегда оцениваются в контексте `companyId`.

## Задачи (что сделать)
- [ ] Расширить текущие интерфейсы ответов агентов (например, `AgentResponse`) массивом `evidence: EvidenceReference[]`.
- [ ] Описать тип `EvidenceReference` (должен содержать `claim` - утверждение, `sourceType` - типа TOOL_RESULT/DB/DOC, `sourceId`, `confidenceScore`).
- [ ] Обновить системный промт (System Prompt) или схему Structured Output у `EconomistAgent` или `KnowledgeAgent`, заставив их выделять `evidenceRefs`.
- [ ] Прокинуть эти `evidenceRefs` через `ResponseComposer` в итоговый ответ/трейс.
- [ ] Написать unit-тесты, проверяющие, что парсер не падает на ответе без evidence и корректно извлекает ссылки на источники, если они есть.

## Definition of Done (DoD)
- [ ] Интерфейсы расширены и компилируются.
- [ ] Выбранный агент (Economist или Knowledge) умеет возвращать привязку утверждений к источнику.
- [ ] Unit-тесты PASS.
- [ ] Сломанных смежных тестов нет.

## Тест-план (минимум)

- [ ] Unit: парсер ответа агента (fallback, где LLM забыла/не смогла отдать evidence).
