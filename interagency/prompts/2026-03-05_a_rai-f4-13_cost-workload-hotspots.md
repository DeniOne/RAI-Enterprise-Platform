# PROMPT — Cost Decomposition & Workload Hotspots (Phase 4.13)
Дата: 2026-03-05
Статус: active
Приоритет: P2

## Цель
Реализовать мониторинг расходов (Cost Decomposition) и выявление самых "горячих" сессий (Workload Hotspots) для Ops-дашборда. Нужно понимать, куда уходят деньги на LLM и какие запросы выполняются дольше всего.

## Контекст
- **Связанные документы:**
  - `docs/00_STRATEGY/STAGE 2/A_RAI_IMPLEMENTATION_CHECKLIST.md` (Phase 4, пункт 4.3: Cost Decomposition & Workload Hotspots).
- У нас уже есть базис трейсов (`TraceSummary` с полями `inputTokens`, `outputTokens`, `totalTimeMs`). Теперь нужно на его основе (и возможно добавить новые модели для DB/внешних апи, если надо) выдавать агрегированную аналитику по стоимости сессий.

## Ограничения
- Фокус на Backend API (Сбор метрик + отдача JSON).
- Изоляция тенантов обязательна (`companyId`). Опс-инженер может по каждому арендатору видеть расходы.
- Стоимость вычисляется "на лету" исходя из токенов (можно задать дефолтные рейты константами в сервисе: например, $2.5 / 1M input, $10.0 / 1M output).

## Задачи (что сделать)
- [ ] Описать `CostAnalyticsService` (или добавить методы в `PerformanceMetricsService`/`TraceSummaryService`):
  - `getTenantCost(companyId, timeWindowMs)` — возвращает общую стоимость LLM по арендатору и разбивку по `agentRole`.
  - `getHotspots(companyId, timeWindowMs, limit = 10)` — возвращает:
    1. Топ-10 самых дорогих трейсов (по сумме токенов/цене).
    2. Топ-10 самых долгих трейсов (по `totalTimeMs`).
- [ ] Опционально: добавить в `TraceSummary` или `AiAuditEntry` поле стоимости запроса к БД/другим API (если это сейчас возможно), либо ограничиться пока LLM Cost.
- [ ] Создать API-эндпоинт (например, `GET /rai/explainability/cost-hotspots`) в `RaiChatController`, который вызывает этот сервис.
- [ ] Покрыть сервис базовыми unit-тестами.

## Definition of Done (DoD)
- [ ] Сервис расчета стоимости и хотспотов реализован.
- [ ] API эндпоинт отдает корректный DTO, соблюдая изоляцию `companyId`.
- [ ] Тесты PASS (`tsc`, `jest`).

- [ ] Unit: Проверка сортировки `getHotspots` (самые дорогие/долгие действительно в топе).
