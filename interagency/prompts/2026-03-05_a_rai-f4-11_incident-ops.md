# PROMPT — Security & Incident Ops (Phase 4.11)
Дата: 2026-03-05
Статус: active
Приоритет: P2

## Цель
Реализовать механизм отслеживания инцидентов безопасности и системных ошибок (`Governance Counters` и `Incidents Feed`). Система должна уметь детектировать кросс-тенантные попытки доступа к чужим агентам/трейсам, срабатывания `SensitiveDataFilterService` (утечки PII) и консолидировать их в ленту инцидентов для мониторинга. 

## Контекст
- **Связанные документы:**
  - `docs/00_STRATEGY/STAGE 2/A_RAI_IMPLEMENTATION_CHECKLIST.md` (Phase 4, пункт 4.5).
- У нас уже есть Tenant Isolation (в `PrismaService` и Context) и `SensitiveDataFilterService`. Теперь нужно считать, **сколько раз** эти защитные механизмы сработали, и кто это сделал. Если агент постоянно пытается обратиться к чужим данным, это либо баг маршрутизации, либо взлом (Prompt Injection).

## Ограничения (жёстко)
- **Изоляция:** Инциденты должны содержать `companyId`, чтобы каждый клиент видел только свои алерты. Глобальные инциденты (без `companyId`) доступны только суперадмину системы (пока реализуем на уровне БД).
- **Производительность:** Счётчики инцидентов не должны блокировать основной флоу. Желательно писать асинхронно или через очередь (или хотя бы огненным `Promise.allSettled`).

## Задачи (что сделать)
- [ ] Добавить в Prisma модель `SystemIncident` (`id`, `companyId` (опционально), `traceId` (опционально), `incidentType` (enum: CROSS_TENANT_BREACH, PII_LEAK, RATE_LIMIT, UNKNOWN), `severity`, `details` (JSON), `createdAt`).
- [ ] Описать сервис `IncidentOpsService`:
  - Метод `logIncident(params)`: записывает инцидент в лог.
  - Метод `getIncidentsFeed(companyId, limit, offset)`: возвращает ленту инцидентов для тенанта с пагинацией.
- [ ] Интегрировать `IncidentOpsService` в `SensitiveDataFilterService`. Когда фильтр реально заменяет текст (нашёл PII), он должен бросать инцидент `PII_LEAK` (уровень LOW/MEDIUM).
- [ ] Интегрировать вызов `PII_LEAK` как тестовый сценарий. (Кросс-тенантные нарушения ловить сложнее, пока можно ограничиться PII и ручными тестами).
