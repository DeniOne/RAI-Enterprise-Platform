Дата: 2026-03-05

## Цель
Реализовать вычисление "критического пути" (Critical Path Analyzer) и топологии вызовов агентов (Agent Connection Map / Retry Topology) внутри трейса. Это необходимо для визуализации графа выполнения сложного запроса (где были ретраи, кто кого ждал, на что реально ушло время).

## Контекст
- **Связанные документы:**
  - `docs/00_STRATEGY/STAGE 2/A_RAI_IMPLEMENTATION_CHECKLIST.md` (Phase 4, пункт 4.3: Agent Connection Map + Critical Path Analyzer).
- У нас есть `AiAuditEntry` с `traceId`, где логируются события (router, fan-out агентов, вызовы tool, composer). Нужно научиться строить по ним граф (дерево) выполнения и агрегировать задержки для поиска самой длинной цепочки (критического пути).

## Ограничения
- Фокус на Backend API (Движок анализа + отдача JSON-графа).
- Изоляция тенантов обязательна (`companyId`). Доступ только к трейсам своей компании.

  - Учтите вложенность (например, Supervisor запустил parallel [Agronom, Economist], каждый вызвал свои tools).
  - Вычислите `durationMs` для каждого узла.

- [ ] Реализовать алгоритм поиска критического пути (Critical Path) — выделить цепочку узлов, чья продолжительность определила общее время ответа (самая долгая ветка в `Promise.allSettled`, плюс последовательные шаги).
