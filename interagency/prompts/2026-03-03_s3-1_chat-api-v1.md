# PROMPT — 3.1 Chat API v1 (Formal Contract)
Дата: 2026-03-03  
Статус: active  
Приоритет: P1  

## Цель
Формализовать контракт Chat API (версия 1), приведя `RaiChatResponseDto` в соответствие со спецификацией Agent OS: добавить поддержку `toolCalls` (список выполненных инструментов) и `openUiToken` (сигнал фронтенду на открытие специфических экранов).

## Контекст (Truth Sync)
- Эндпоинт: `POST /api/rai/chat`.
- Файлы: `apps/api/src/modules/rai-chat/dto/rai-chat.dto.ts`, `apps/api/src/modules/rai-chat/rai-chat.service.ts`.
- В текущей реализации `RaiChatResponseDto` отсутствуют поля `toolCalls` и `openUiToken`, хотя сервис уже умеет исполнять инструменты.

## Задачи

### 1. Расширение DTO (apps/api)
- **RaiChatResponseDto**:
  - Добавить `@IsArray()`, `@IsOptional()`, `@ValidateNested({ each: true })`, `@Type(() => RaiToolCallDto)` для поля `toolCalls?: RaiToolCallDto[]`.
  - Добавить `@IsString()`, `@IsOptional()`, `@MaxLength(128)` для поля `openUiToken?: string`.
- Убедиться, что все импорты и декораторы `class-validator`/`class-transformer` на месте.

### 2. Реализация в сервисе (apps/api)
- Обновить `handleChat` в `RaiChatService`:
  - Собирать результаты выполненных инструментов (имена + пэйлоады вызовов) в массив `RaiToolCallDto[]`.
  - Включать этот массив в объект `RaiChatResponseDto`.
  - Подготовить структуру для будущего использования `openUiToken` (пока оставить опциональным/null, если нет условий для активации).

### 3. Верификация
- Обновить или добавить тест в `src/modules/rai-chat/rai-chat.service.spec.ts`, проверяющий, что:
  - В ответе `RaiChatResponseDto` присутствует поле `toolCalls` (если инструменты вызывались).
  - Поле `traceId` и `threadId` корректно возвращаются (уже должно работать, просто подтвердить).
- Выполнить `tsc` для `apps/api`, чтобы исключить ошибки типов.

## Definition of Done
- [ ] `RaiChatResponseDto` содержит `toolCalls` и `openUiToken`.
- [ ] `RaiChatService` возвращает список выполненных инструментов.
- [ ] Тесты `rai-chat.service.spec.ts` проходят (учитывая новые поля).
- [ ] `apps/api` проходит `tsc`.
- [ ] Подготовлен отчёт по канону Software Factory.

## Запреты
- НЕ менять контракт `workspaceContext` (он уже зафиксирован в S2.1/S2.2).
- НЕ менять логику `companyId` (строгая изоляция тенанта).
- НЕ реализовывать сложную логику для `openUiToken`, достаточно просто добавить поле в контракт.
