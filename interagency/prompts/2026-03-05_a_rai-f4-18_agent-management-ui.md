# PROMPT — Agent Registry & Configurator UI (Phase 4.18)
Дата: 2026-03-05
Статус: active
Приоритет: P2

## Цель
Реализовать пользовательский интерфейс для управления реестром агентов. Это позволит настраивать поведение роя (промпты, модели, доступы) прямо из интерфейса управления.

## Контекст
- **Backend API:** `CRUD /rai/agents/config` (согласно F4.16).
- **Схема:** `AgentConfiguration` (name, role, systemPrompt, llmModel, capabilities, isActive).

## Задачи (что сделать)
- [ ] **Список агентов (Agent Registry):**
  - Таблица/карточки со статусом (Active/Inactive), ролью и текущей моделью (GPT-4/Claude и т.д.).
  - Быстрое переключение `isActive`.
- [ ] **Редактор конфигурации (Agent Configurator):**
  - Форма редактирования `systemPrompt` (режим Monaco Editor или TextArea с моноширинным шрифтом).
  - Выбор модели из списка.
  - Управление `capabilities`: мультиселект доступных реестров/инструментов.
- [ ] **Tenant Overrides:**
  - Возможность создавать специфические настройки для конкретного `companyId`.
  - Индикация, является ли конфиг глобальным или переопределённым тенантом.

## Стек и стиль
- Тот же, что и в F4.17. Премиальный UI, удобство редактирования длинных промптов.

## Definition of Done (DoD)
- [ ] Полный цикл CRUD для настроек агентов через UI.
- [ ] Tenant isolation соблюдена.
- [ ] Сборка веба проходит.
