# PLAN — AppShell с персистентным RAI Chat
Дата: 2026-03-02  
Статус: draft  
Decision-ID: AG-APP-SHELL-001  

## Результат (какой артефакт получим)
- Исполнимый план для `S1.1`: вынести RAI Chat в глобальный `AppShell` в `apps/web`, чтобы чат жил в shell-layout и не размонтировался при переходах между страницами.
- Зафиксированный минимальный frontend-scope: только shell-обёртка, точка монтирования чата и точечные правки route-change/state persistence без переписывания продуктовых роутов.
- Явное соответствие требованию stage-plan: история чата, режимы `Dock/Focus` и связанный UI-state сохраняются при навигации.

## Границы (что входит / что НЕ входит)
- Входит: анализ текущей точки монтирования `AiChatPanel`, логики закрытия на `ROUTE_CHANGE`, связки со store и поведения `AiChatWidgetsRail`.
- Входит: введение минимального `AppShell` layout в `apps/web` с зонами `TopNav`, `LeftRaiChatDock` и `MainWorkspace` без редизайна продуктовых экранов.
- Входит: перенос `AiChatPanel` в shell-контур так, чтобы компонент оставался в DOM при смене роутов и продолжал использовать существующий store/localStorage.
- Входит: проверка того, что `AiChatWidgetsRail` продолжает рендериться и не блокирует основной workspace.
- Не входит: backend-изменения, изменение API `/api/rai/chat`, правки `WorkspaceContext`, переписывание страниц/роутов сверх нужной shell-интеграции.
- Не входит: новый визуальный язык, анимации, декоративная полировка, тяжёлый layout-refactor вне задачи персистентного shell.

## Риски (что может пойти не так)
- Есть риск, что текущая точка монтирования чата завязана на страничный layout или overlay-state, и перенос в shell сломает существующие триггеры открытия/закрытия.
- Есть риск регрессии `Dock/Focus`, если route-change cleanup сейчас неявно сбрасывает состояние, которое также влияет на shell-layout.
- Есть риск конфликтов с правым rail, если новый shell изменит stacking, ширины колонок или события фокуса.
- Есть риск ложного “исправления”, при котором сохранится store-state, но сам React subtree будет всё равно размонтироваться и терять локальный UI-state.

## План работ (коротко, исполнимо)
- [ ] Найти фактическую точку монтирования `AiChatPanel` и определить, что именно вызывает закрытие или размонтирование при `ROUTE_CHANGE`.
- [ ] Найти текущий layout/root entrypoint в `apps/web`, куда можно ввести минимальный `AppShell` без переписывания продуктовых страниц.
- [ ] Спроектировать shell-структуру `TopNav + LeftRaiChatDock + MainWorkspace` с минимальным влиянием на существующий UI и без нарушения текущего дизайн-канона.
- [ ] Переместить `AiChatPanel` в `LeftRaiChatDock` так, чтобы subtree чата оставался смонтированным при навигации.
- [ ] Уточнить и исправить логику auto-close на route-change: убрать безусловное закрытие либо ограничить его сценариями, которые не конфликтуют с `Docked` режимом.
- [ ] Проверить, что `Dock/Focus` продолжает храниться и восстанавливаться через текущий persistence-механизм.
- [ ] Проверить совместимость с `AiChatWidgetsRail` и основным workspace на нескольких страницах.
- [ ] После реализации прогнать `apps/web` typecheck, существующие чатовые тесты и manual smoke по навигации.

## DoD
- [ ] При переходах между страницами чат не размонтируется, история сообщений и UI-состояние сохраняются.
- [ ] `Dock/Focus` работает как раньше и персистится без сброса на навигации.
- [ ] `AiChatWidgetsRail` не регрессирует и не блокирует `MainWorkspace`.
- [ ] `cd apps/web && pnpm exec tsc -p tsconfig.json --noEmit` проходит.
- [ ] `cd apps/web && pnpm test -- --runInBand __tests__/ai-chat-store.spec.ts __tests__/ai-chat-widgets-rail.spec.tsx` проходит.
