# Бриф для сбора исторических данных для обучения ядра ИИ

## CLAIM
id: CLAIM-DOM-AGRO-TECHMAP-AI-CORE-HISTORICAL-DATA-BRIEF-20260402
status: asserted
verified_by: manual
last_verified: 2026-04-02

## Назначение

Этот документ задаёт, какие исторические данные нужны не для одного хозяйства, а для обучения и калибровки ядра ИИ.

Цель:
- собрать данные, на которых ядро сможет учиться закономерностям;
- отделить индивидуальную историю хозяйств от универсального обучающего корпуса;
- понять, какие данные должны приходить из институтов, статистических служб, региональных управлений АПК, гидрометеорологии, сортоиспытаний, агрохимслужбы и других официальных источников;
- сформировать требования к качеству, полноте, происхождению и пригодности данных для обучения.

## Для чего эти данные нужны ядру ИИ

Ядру ИИ нужны не только успешные техкарты.

Ему нужны данные, чтобы учиться:
- различать хорошие и плохие сценарии;
- понимать связь между зоной, почвой, погодой, технологией и урожаем;
- калибровать нормативы и ожидания по урожаю, рискам и срокам;
- отделять системные закономерности от локальных частных кейсов;
- строить explainable рекомендации, а не только угадывать ответы.

## Главный принцип

Для обучения ядра ИИ нужны четыре класса исторических данных:

1. Нормативные и научные данные.
2. Массовая агрегированная статистика.
3. Региональные операционные и агрометеоданные.
4. Реальные обезличенные данные хозяйств и полевых наблюдений.

Только комбинация этих четырёх классов даёт устойчивое ядро.

## Приоритетные источники

Источники нужно собирать по группам.

### 1. Профильные институты и НИИ

Нужны данные:
- полевые опыты;
- сортоиспытания;
- агрохимические нормы;
- зависимости урожая от питания, сроков сева, густоты стояния, защиты;
- результаты по болезням, вредителям и стрессам;
- методики и пороги вредоносности;
- зональные рекомендации.

### 2. Статистика

Нужны данные:
- посевные площади;
- урожайность по культурам;
- динамика урожайности по регионам;
- динамика площадей;
- валовой сбор;
- структура культур;
- погодные аномалии и ущерб, если присутствуют в статистике.

### 3. Региональные управления АПК и отраслевые службы

Нужны данные:
- состояние озимых;
- сводки по срокам сева и уборки;
- мониторинг вредителей и болезней;
- отчёты по ЧС, засухам, вымерзанию, подтоплению;
- районные технологические окна;
- оперативные сводки сезона.

### 4. Гидрометеорология и агрометеорология

Нужны данные:
- температура;
- осадки;
- засушливые периоды;
- заморозки;
- снежный покров;
- дата схода снега;
- накопление тепла;
- аномалии и экстремальные события;
- временные ряды по декадам, неделям и месяцам.

### 5. Агрохимслужба, почвенные и картографические источники

Нужны данные:
- карты почв;
- агрохимические обследования;
- кислотность;
- обеспеченность элементами питания;
- органическое вещество;
- ограничения по почвам;
- эрозия, засоление, переувлажнение, уплотнение, водный режим.

### 6. Сортоиспытание и селекционные данные

Нужны данные:
- результаты сортов и гибридов по зонам;
- устойчивость к болезням;
- зимостойкость;
- скороспелость;
- отклик на интенсивность;
- качество продукции;
- стабильность по годам.

## 9. Данные для причинного анализа

Ядру ИИ особенно важны пары вида:
- условие;
- действие;
- результат.

Нужны таблицы и кейсы, где можно связать:
- какая была почва;
- какая была погода;
- какую операцию провели;
- какой продукт и дозу использовали;
- какой был фактический результат по культуре;
- что случилось с урожаем и качеством.

Без этого ядро учится корреляциям, но плохо учится причинности.

## 10. Какие данные особенно ценны из внешних источников

### Самые ценные для базового обучающего ядра

- многолетние ряды урожайности по районам и зонам;
- многолетние погодные ряды;
- результаты сортоиспытаний по регионам;
- официальные сводки по перезимовке, болезням и вредителям;
- агрохимические карты и обследования;
- многолетние полевые опыты по нормам высева, срокам сева, удобрениям, защите;
- обезличенные field-level наборы из хозяйств, где есть связка `технология -> факт -> урожай`.

### Особенно ценные для рапса

- данные по перезимовке озимого рапса;
- response на серу и бор;
- данные по влиянию сроков сева;
- данные по густоте стояния и развитию розетки;
- данные по осыпанию и срокам уборки;
- данные по склеротиниозу, альтернариозу, блошкам, скрытнохоботникам и цветоеду;
- данные по килам и ограничениям по `pH`.

## 11. Обязательные требования к каждому набору данных

Каждый источник для обучения должен иметь:
- происхождение;
- право использования;
- дату охвата;
- географический охват;
- уровень агрегации;
- единицы измерения;
- описание пропусков;
- описание методики сбора;
- версию и дату выгрузки.

## 12. Что нельзя брать в обучающий контур без контроля

- данные без подтверждённого источника;
- статьи и презентации без исходных чисел;
- таблицы без единиц измерения;
- агрегаты, где неизвестно, как они считались;
- данные с неясным правом использования;
- кейсы, где нельзя отличить факт от экспертного мнения;
- непроверенные пользовательские наблюдения без минимального контекста.

## 13. Рекомендованная структура корпуса для ядра ИИ

| Слой корпуса | Что содержит | Назначение |
|---|---|---|
| `Normative Layer` | научные нормы, пороги, методики, регламенты | канонические правила и ограничения |
| `Statistical Layer` | многолетняя статистика по регионам и культурам | базовые ожидания и калибровка |
| `Agrometeo Layer` | погодные и агрометеорологические ряды | климатический и сезонный контекст |
| `Trial Layer` | опыты, сортоиспытания, response-таблицы | причинные зависимости |
| `Operational Layer` | обезличенные реальные истории хозяйств | проверка применимости в поле |
| `Observation Layer` | наблюдения, фото, сигналы, алерты | runtime-контур и explainability |

## 15. Критический минимум для старта обучения ядра

Если собирать ядро поэтапно, стартовый обязательный набор такой:

- статистика урожайности по культурам и регионам за 5–10 лет;
- погодные ряды за тот же период;
- результаты сортов и гибридов по зонам;
- агрохимические и почвенные характеристики по зонам;
- нормативы и response-данные по срокам сева, питанию и защите;
- обезличенные полевые истории минимум по части хозяйств;
- корпус наблюдений по болезням, вредителям, погодным стрессам и перезимовке.

## 16. Практический вывод

Для ядра ИИ важны не просто «данные из институтов».

Нужна обучающая система данных, в которой:
- есть зона и время;
- есть почва и погода;
- есть технология и исполнение;
- есть техника и ресурсные ограничения;
- есть факт урожая и качества;
- есть причинные связи и объяснимость.

Только на таком корпусе можно обучать ядро, которое будет не пересказывать общие советы, а строить устойчивые и объяснимые техкарты.
