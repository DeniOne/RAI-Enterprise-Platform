# Бриф для сбора исторических данных от хозяйства

## CLAIM
id: CLAIM-DOM-AGRO-TECHMAP-HISTORICAL-FARM-DATA-BRIEF-20260402
status: asserted
verified_by: manual
last_verified: 2026-04-02

## Назначение

Этот документ используется для сбора исторических данных от конкретного хозяйства.

Цель:
- собрать историю, на которой можно делать индивидуальный агрономический анализ;
- сформировать индивидуальную базу хозяйства без потери контекста по полям, технике, погоде и исполнению;
- получить материал для сравнения `план -> факт -> причина -> результат`;
- создать основу для следующих циклов разработки техкарт, прогнозов урожайности, рекомендаций и анализа отклонений.

## Для чего нужна историческая база хозяйства

Историческая база нужна не только для хранения прошлых урожаев.

Она должна позволять ответить на вопросы:
- что реально происходило на каждом поле по сезонам;
- какие технологии применялись;
- какая техника и с какими ограничениями использовалась;
- где были погодные, почвенные, организационные и ресурсные ограничения;
- какие факторы реально влияли на урожай, качество и себестоимость;
- какие решения повторяются как успешные, а какие как неуспешные.

## Горизонт истории

Рекомендуемый минимум:
- 3 завершённых сезона;
- желательно 5 сезонов;
- если есть архив глубже, собирать 7–10 лет по урожаю, погоде, культуре, агрохимии и ключевым технологическим решениям.

## Правило единицы хранения

Исторические данные должны собираться минимум на уровне:
- хозяйство;
- сезон;
- поле;
- культура на поле;
- операция;
- факт выполнения;
- урожай и качество;
- техника и ресурсное ограничение;
- событие риска или отклонения.

## Блок критического минимума

Если хозяйство не может дать всё, нужно в первую очередь собрать:
- историю культур по полям;
- историю урожайности по полям;
- историю основных операций;
- историю агрохимии и анализов почвы;
- историю погодных аномалий и критических событий;
- историю техники и доступных технологических окон;
- историю затрат по операциям и материалам;
- историю проблем: сорняки, болезни, вредители, перезимовка, пересев, простои.

## Что особенно важно для индивидуального анализа хозяйства

Наибольшую ценность дают:
- полевая история по сезонам;
- история урожайности именно по полю, а не только в среднем по хозяйству;
- история севооборота;
- история сроков и факта операций;
- история техники и простоев;
- история дефицитов ресурсов;
- история погодных стрессов;
- история перезимовки и пересевов;
- история экономики по полю.

## Структура индивидуальной базы хозяйства

Историческая база должна позволять связать между собой:
- поле;
- сезон;
- культуру;
- фактические операции;
- применённые материалы;
- технику;
- погодные события;
- отклонения;
- урожай;
- экономику.

Без этой связности историческая база превращается в архив таблиц, а не в рабочую основу для анализа и следующих техкарт.

## Минимальные статусы полноты

| Статус | Смысл |
|---|---|
| `MINIMUM_HISTORY_READY` | Есть история культур, урожайности, операций и ключевых почвенных данных минимум за 3 сезона. |

| `FULL_INDIVIDUAL_BASE_READY` | Есть связная история поле × сезон × операция × техника × ресурс × урожай × экономика. |
