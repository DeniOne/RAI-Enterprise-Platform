# Бриф хозяйства для разработки техкарты

## CLAIM

## Назначение

Этот бриф используется для первичного сбора данных от хозяйства перед разработкой техкарты.

Цель брифа:
- собрать один пакет входных данных без многократных уточнений;
- отделить обязательные данные для генерации от желательных уточнений;
- сразу выявить пробелы, которые блокируют допуск поля и построение техкарты;
- унифицировать вход для агронома, аналитика и команды генерации.

## Когда применять

- при первом создании техкарты по полю;
- при регенерации техкарты на новый сезон;
- при переводе хозяйства на каноническую генерацию техкарт;
- при ручной разработке техкарты без автоматической генерации, чтобы не потерять обязательные входы.

## Правило заполнения

- один бриф заполняется на одно хозяйство;
- приложение по полям заполняется отдельно на каждое поле или однородный контур;
- если данные неизвестны, поле не оставляют пустым, а помечают как `нет данных` или `требует замера`;
- для всех числовых показателей обязательно указывать единицу измерения и дату источника;
- для всех лабораторных и картографических данных обязательно прикладывать источник: файл, протокол, карту, ссылку, скан.

## Критический минимум для старта разработки техкарты рапса

Без этих данных техкарта рапса считается неполной и должна идти в режим `PASS_WITH_REQUIREMENTS` или блокироваться до уточнения:

- форма культуры: озимый или яровой рапс;
- поле и сезон;
- агроклиматическая зона;
- `SAT_avg`;
- гранулометрический состав почвы;
- `pH`;
- `P_available`;
- `K_available`;
- `S_available`;
- `B_available`;
- предшественник;
- лет без рапса в севообороте;
- история килы;
- наличие уплотнения;
- сорт или гибрид;
- целевая урожайность.

## Правило интерпретации брифа

- заполненный бриф не равен готовой техкарте;
- бриф является входом для допуска поля, выбора ветки культуры и статической адаптации;
- если в брифе не хватает критических полей, команда обязана зафиксировать это как явный дефицит данных, а не додумывать показатели;
- данные уровня региона и данные уровня поля запрещено смешивать в один источник.

## Отдельное семантическое правило

- `RegionProfile` содержит региональные и агроклиматические данные: `SAT_avg`, зона, тип зимы, риски засухи и заморозков;
- `SoilProfile` содержит локальные полевые и почвенные данные: `pH`, `P_available`, `K_available`, `S_available`, `B_available`, уплотнение, дренаж, свежесть анализа.

Это разграничение обязательно для корректной разработки техкарты.
