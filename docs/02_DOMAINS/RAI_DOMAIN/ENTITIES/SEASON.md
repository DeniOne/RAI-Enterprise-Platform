# Entity: Season (Сезон) 📅

> **Статус:** Канон | **Версия:** 1.1 | **Слой:** RAI/Entities

---

## 1. Определение
**Сезон (Season)** — это центральный управляющий объект в домене RAI. Он объединяет Поле, Культуру и временной период в единый производственный проект.

---

## 2. Жизненный цикл (Statuses)

Мы используем `isLocked` механизм для обеспечения неизменяемости истории:

1. **PLANNED**: Идет подготовка, выбор техкарты, закупка ресурсов.
2. **ACTIVE**: Культура в поле. Идет выполнение задач.
3. **COMPLETED**: Урожай собран. Данные проверены.
4. **LOCKED**: Сезон заархивирован. Изменения запрещены. Создан `SeasonSnapshot`.

---

## 3. Бизнес-правила (Hardening)

В системе реализованы "железные" правила валидации (см. `SeasonBusinessRulesService`):

### 3.1 Севооборот (Crop Rotation)
Запрещено создавать сезон с `Рапсом`, если на этом поле он уже рос в течение последних 3 лет. Система проверяет историю через `Registry`.

### 3.2 Даты (Agronomic Windows)
Даты посева должны соответствовать типу культуры:
- Озимый рапс: Август - Сентябрь.
- Яровой рапс: Апрель - Май.

---

## 4. Снапшоты (Data Integrity)
При переходе в статус `LOCKED` система делает полный слепок состояния сезона в таблицу `SeasonSnapshot`. Это гарантирует, что даже если справочники изменятся (например, изменится описание сорта рапса), история конкретного урожая останется достоверной.

---

## 5. Связи
- **Field**: Одно поле — один активный сезон.
- **Crop**: Ссылка на биологический вид и сорт.
- **Tasks**: Все задачи по техкарте привязаны к ID сезона.
- **Audit**: Каждая смена статуса сезона фиксируется в аудите с подписью ответственного.
