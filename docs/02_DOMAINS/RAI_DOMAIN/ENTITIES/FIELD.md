# Entity: Field (Поле) 🗺️

> **Статус:** Канон | **Версия:** 1.0 | **Слой:** RAI/Entities

---

## 1. Определение
**Поле (Field)** — ключевая элементарная единица земельного фонда компании. Это не просто кусок земли, а актив с уникальной геометрией, физико-химическими свойствами и историей эксплуатации.

---

## 2. Модель данных (Спецификация)

### 2.1 Атрибуты (Core)
- `id` (UUID): Уникальный идентификатор.
- `companyId`: Привязка к владельцу (Multi-tenancy).
- `name`: Название или номер поля (напр., "Полоса-С").
- `area`: Общая площадь в гектарах (заполняется автоматически по геометрии).
- `legalArea`: Площадь по документам.

### 2.2 Геометрия (PostGIS)
- `polygon`: Геометрический тип `POLYGON` (WGS 84).
- `center`: Центроид поля (для отображения маркера на карте).

---

## 3. Бизнес-логика
1. **Валидация границ**: Поля одной компании не должны пересекаться более чем на 5% (допуск на погрешность GPS).
2. **Авто-расчет**: При изменении геометрии площадь `area` должна пересчитываться мгновенно.
3. **Жизненный цикл**: Поле не может быть удалено, если на нем есть активный или завершенный `Season`. Только перевод в статус `ARCHIVED`.

---

## 4. Связи
- **Registry Node**: Поле является узлом в оргструктуре хозяйства.
- **Season**: Одно поле в один момент времени может иметь только один активный `Season` (исключение: промежуточные культуры).
- **Soil Analysis**: История анализов почвы привязана к геометрии поля.

---

## 5. Представление в UI
- **Map View**: Отрисовка полигона с заливкой. Цвет заливки зависит от текущей культуры или вегетации (NDVI).
- **Statistics Widget**: Краткая сводка: площадь, текущая культура, активные задачи.

---

## 6. API Контракты
- `searchFields(bounds: BBox)` — поиск полей в видимой области карты.
- `calculateIntersection(polygon: GeoJSON)` — проверка наложение на существующие поля.
