# Подробный чек-лист фазы Gamma (12 недель)

Этот документ разжевывает каждый спринт «на пальцах», чтобы даже твоя бабушка (или очень занятой босс) поняла, за что мы платим деньги и че там за магия происходит.

---

## 🏗️ Блок 1: Фундамент (Foundation)

### Спринт №1: «Цифровая гигиена и переезд на K8s»
**Цель:** Подготовка среды + данные + наблюдаемость для AI-продуктов.
- [x] **Kubernetes (K8s) Setup:** namespace `rai-gamma`, лимиты ресурсов и квоты.
- [ ] **Secrets Management:** перенос секретов в K8s Secrets.
- [x] **Trace ID Propagation:** сквозная передача trace-id (OTel-compatible) на всех операциях.
- [x] **Data Contracts:** унифицированные JSON-схемы (semver), хранение в Git.
- [x] **pgvector Smoke Test:** проверка записи/чтения векторов.
- [x] **Ingestion Scaffolding:** костяк воркеров для асинхронного поглощения сигналов.
- [x] **Risk Engine Dry-Run Hook:** read-only endpoint для ГАММЫ.

### Спринт №2: «Запуск Семантического Графа и Baseline ИИ»
**Цель:** Дать системе базовое зрение и зачатки памяти (**Baseline Readiness**).
**Статус:** завершен (08.02.2026)
- [x] **Knowledge Graph MVP:** Загружаем связи (Поле -> Культура -> Сорняк). Система начинает понимать "кто с кем спит" в агро-мире.
- [x] **Vision AI Baseline:** Запуск нейронки в режиме «источник сигнала». Различает базу, но не дает советов.
- [x] **Satellite Ingestion:** Подключаем поток данных со спутников (NDVI). Калибровка и проверка стабильности сигнала.
- [x] **Vector Storage:** Поднимаем специальную «память» (pgvector). Это база, где ИИ будет хранить свой опыт в виде цифр (векторов).

> [!NOTE]
> Спринт №2 — это **источник сигнала**, а не работающий механизм рекомендаций. Бизнес-ценность на этом этапе минимальна, фокус на стабильности данных.

## 🧠 Блок 2: Интеграция (Integration)

### Спринт №3: «Когнитивная память (Уроки прошлого)»
**Цель:** Научить ИИ вспоминать: «Слышь, мы тут в прошлом году уже косячили».
- [x] **Episodic Retrieval:** Базовый retrieval-сервис и unit-тесты реализованы (Shadow v1).
- [x] **Positive/Negative Engrams:** Базовые правила исходов вынесены в `engram-rules` + unit-тесты.
- [x] **Advisory Logic:** Реализован `ShadowAdvisoryService` с ранжированием `ALLOW/REVIEW/BLOCK`.
- [x] **Shadow Mode Start:** Теневой advisory-контур активирован на уровне сервиса, без UI-публикации.

### Спринт №4: «Движок объяснений и UX-интеграция»
**Цель:** Сделать ИИ понятным человеком и вывести его в Телеграм/Веб.
**Детальный план:** `docs/06-IMPLEMENTATION/PHASE_GAMMA/SPRINT_4_CHECKLIST.md`
- [x] **Explainability Module:** Учим ИИ объяснять. Блок «Почему я так решил» (т.к. влажность 80%, а в прошлом году это привело к грибку).
- [x] **Recommendation Cards:** Рисуем карточки в Телеграм-боте с кнопками «Принять/Отклонить».
- [x] **Feedback Loop:** Делаем механизм сбора пиздюлей от агрономов. Если он отклонил совет — система записывает почему.
- [x] **Confidence Score:** Добавляем индикатор уверенности: "Я уверен на 90%" или "Хрен его знает, на 30%, лучше проверь сам".

## 🚀 Блок 3: Пилот и Запуск (Pilot & Hardening)

### Спринт №5: «Живой пилот и тюнинг»
**Цель:** Выпустить ИИ в реальное поле под присмотром доверенных людей.
**Детальный план:** `docs/06-IMPLEMENTATION/PHASE_GAMMA/SPRINT_5_CHECKLIST.md`
- [x] **Incident Runbook:** Тренируемся гасить ИИ, если он вдруг начнет советовать залить поле бензином (защитные контуры).
- [x] **Pilot Activation:** Раскатываем доступ для фокус-группы агрономов.
- [x] **Ranking Tuning:** Исправляем назойливость. Чтобы ИИ не спамил тупыми советами каждые 5 минут.
- [x] **SLO Dashboard:** Делаем приборку для админов: как быстро ИИ отвечает и не отваливаются ли коннекты.

### Спринт №6: «Hardening и Поехали!»
**Цель:** Сделать систему пуленепробиваемой и запуститься на всех.
**Детальный план:** `docs/06-IMPLEMENTATION/PHASE_GAMMA/SPRINT_6_CHECKLIST.md`
- [x] **Canary Rollout:** staged rollout-контур и rollback/auto-stop проверки выполнены в pilot rehearsal.
- [x] **Load Testing:** baseline + stress-профили advisory пройдены, SLO подтверждены.
- [x] **Final DoD Check:** критерии Sprint 6 закрыты, зафиксирован formal go/no-go record.
- [x] **Go-Live:** Controlled launch выполнен до `S3` (50%) по результатам stage progression и formal `GO` decision.
