# UI Design: Agro-Tech Map Builder (АТК)

> **Модуль:** Технологическая карта (АТК)  
> **Цель:** Инструмент для агроконсультанта по созданию сверхдетального плана.

---

## 1. Конструктор АТК (Builder UI)

### 1.1. Рабочая область (The Canvas)
- **Левая панель**: Список всех 16 этапов (хронология).
- **Центр**: Таймлайн операций выбранного этапа. 
- **Правая панель**: Свойства (Property Grid) выбранной операции.

### 1.2. Визуализация почасового регламента
- Каждое действие (посев, внесение гербицида) отображается в виде слота на временной шкале.
- Использование `Geist Mono` для отображения точного времени (08:00 - 10:00).
- Цветовая кодировка типов работ: Посев (Black), Защита (Red), Питание (Green).

---

## 2. Интеграция АТК в CMR (Consulting View)

В CMR карта АТК не редактируется, она **мониторится**.

### 2.1. Экран "Карточка проекта — План vs Факт"
- **Верхний слой**: Эталонная АТК (то, что запланировал консультант).
- **Нижний слой**: Реальная лента событий из поля.
- **Индикация**: Если операция выполнена с отклонением по времени или регламенту — загорается `Risk Card` (⚠️).

### 2.2. Decision Log на базе АТК
- Автоматическая запись: "Операция 'Внесение' выполнена с опозданием на 4 часа. ИИ пересчитывает риски болезней".

---

## 3. Мобильный интерфейс (Для поля — Telegram)
- Полевой работник получает из АТК только "Задачу на сейчас".
- Детализация: "Скорость 6-8 км/ч, глубина 2 см, расход 200 л/га".
- Подтверждение: Фото-фиксация или кнопка "Готово".
