# Knowledge Fabric UI: Operational Intelligence 🕸️

> **Цель**: Операционная полезность, навигация по смыслам и обнаружение архитектурных пробелов.
> **Принцип**: Граф как инструмент мышления, а не просто визуализация.

## 1. Концепция: Три смысловых вектора

Фронтенд Knowledge Fabric не является заменой документации или простым UI. Это поисковая машина над графом знаний проекта:

1.  🧭 **Навигация по смыслу**: Переход от высокоуровневых принципов к конкретным метрикам и коду.
2.  🔍 **Обнаружение пробелов**: Автоматическая подсветка "слепых зон" (блоки без метрик, решения без ответственных).
3.  🧠 **Стратегическое мышление**: Визуализация зависимостей для оценки влияния изменений (Impact Analysis).

---

## 2. Frontend Prompt (Спецификация для реализации)

### Входные данные (Input)
- `graph.json`: Объект с типизированными узлами (`principle`, `control`, `metric`, `component`, `decision`) и ребрами (`aligned_with`, `measured_by`, `depends_on`).
- `Markdown Source`: Рендеринг исходных документов при выборе узла.

### Требования к функционалу
1.  **Исследовательский фокус**: Граф — вторичный вид. Основной фокус на данных узла.
2.  **Глубокая фильтрация**: Сортировка по `type`, `status`, `owner`, `layer`.
3.  **Инспектор узла (Node Inspector)**:
    - Метаданные (YAML-фронтматер).
    - Входящие/Исходящие связи (Inbound/Outbound).
    - Прямые ссылки на файлы.
4.  **Семантические запросы (Semantic Queries)**:
    - `Unmeasured Principles`: Принципы без привязанных метрик.
    - `Enforced Without Control`: Компоненты со статусом `enforced`, но без `control`.
    - `Metrics Without Consumers`: Заброшенные метрики.
    - `Risks Without Mitigation`: Решения с высоким риском без компенсирующих мер.
5.  **Timeline View**: История ревью на основе `last_reviewed`.
6.  **Layouts**:
    - `Hierarchical`: Сверху вниз (Principle → Control → Metric).
    - `Force-Directed`: Группировка по доменам/слоям.

### Ограничения (Non-goals)
- Никакого свободного редактирования графа мышкой.
- Никакого WYSIWYG редактирования маркдауна.

## 3. MVP UI Components

### 📊 Query Panel
Панель предопределенных "инженерных фильтров" для быстрого аудита базы знаний.

### 🧩 Node Inspector
Сайдбар, открывающийся при клике на узел. Показывает "карточку смысла": ID, Владелец, Текст документа.

### 🕸 Graph View
Интерактивное полотно (на базе `react-force-graph` или `vis.js`), визуализирующее связи между сущностями.

### 🚨 Violations View
Прямая интеграция с `ontology_check.py` и CI: список нарушений онтологии в реальном времени.

## 4. Философия Knowledge Fabric

Ты не внедряешь «цифровой сад». Ты строишь **Knowledge Fabric**, где:
- **Документация** = Индекс реальности.
- **Код** = Исполнение принципов.
- **Граф** = Инструмент мышления.
