# ДОМЕННАЯ ОНТОЛОГИЯ РАПСА
## Канонический словарь домена и семантический контракт для подсистеми техкарт рапса

**Версия:** 1.0.0  
**Домен:** Рапс (Brassica napus L.) — Озимый и яровой  
**Исходные модули:** M1–M5  
**Сгенерировано:** 2026-04-01  
**Статус:** Канонический — официальный источник для платформы RAI_EP

---

## 1. Назначение документа

Этот документ определяет **официальные сущности, концепции, перечисления и семантические границы** для домена технологических карт рапса в RAI_EP.

Он служит как:
- **Семантический контракт** между разработчиками и агрономами (доменными экспертами)
- **Авторитетный источник именования** объектов в коде, базе данных, API и документации
- **Справочник для устранения неоднозначностей** в общих агрономических концепциях
- **Карта взаимосвязей** для понимания потоков данных и зависимостей сущностей

Все реализации (схемы баз данных, интерфейсы TypeScript, API-эндпоинты) ДОЛЖНЫ соответствовать этой онтологии.

## 2. Ключевые сущности домена (Core Domain Entities)

### 2.1. Поле (Field)

| Атрибут | Значение |
|-----------|-------|
| **canonical_name** | `Field` |
| **description** | Конкретный физический земельный участок с устойчивыми географическими и почвенными характеристиками. Фундаментальная единица планирования. |
| **why_it_exists** | Каждая техкарта привязана к полю. Характеристики поля определяют логику статической адаптации. |
| **key_attributes** | `field_id` (UUID), `coordinates` (lat/lon), `altitude` (m), `area_ha`, `farm_id` |
| **relationships** | Имеет один `SoilProfile`, один `AgroclimaticProfile`, множество `CropPlan` и `SeasonalObservation`. |
| **lifecycle_position** | Персистентный; создается один раз, обновляется редко. |
| **mandatory** | ДА |

### 2.2. План посева (CropPlan)

| Атрибут | Значение |
|-----------|-------|
| **canonical_name** | `CropPlan` |
| **description** | Сезонный план выращивания рапса на конкретном поле. Связывает поле, форму культуры, сорт и целевую урожайность. |
| **why_it_exists** | Оркестрирующая сущность, которая инициирует генерацию техкарты и отслеживает выполнение сезона. |
| **key_attributes** | `crop_plan_id`, `field_id`, `season_year`, `crop_form` (Enum), `variety_id`, `target_yield` (t/ha), `rotation_years_since_rapeseed`. |
| **relationships** | Принадлежит одному `Field`. Имеет одну `RapeseedTechCard`, программы питания и защиты. |
| **lifecycle_position** | Создается на сезон. Активен во время сезона, архивируется после уборки. |
| **mandatory** | ДА |

### 2.3. Технологическая карта (RapeseedTechCard)

| Атрибут | Значение |
|-----------|-------|
| **canonical_name** | `RapeseedTechCard` |
| **description** | Сгенерированная и адаптированная дорожная карта возделывания рапса. Состоит из последовательных этапов с операциями и точками контроля. |
| **why_it_exists** | Основной артефакт системы. Адаптированный экземпляр канонического технологического шаблона. |
| **key_attributes** | `techcard_id`, `crop_plan_id`, `crop_form`, `version`, `status` (Enum), `stages[]`. |
| **relationships** | Принадлежит `CropPlan`. Содержит упорядоченные `TechCardStage`. |
| **lifecycle_position** | Генерируется один раз за сезон; может быть обновлена при изменении ключевых параметров. |
| **mandatory** | ДА |

### 2.4. Этап техкарты (TechCardStage)

| Атрибут | Значение |
|-----------|-------|
| **canonical_name** | `TechCardStage` |
| **description** | Логическая фаза внутри техкарты, представляющая целостный агрономический блок (например, "Уборка", "Стеблевание"). |
| **why_it_exists** | Организует технологию в управляемые блоки. Каждый этап имеет цель, операции и точку контроля. |
| **key_attributes** | `stage_id`, `techcard_id`, `order`, `name`, `goal`, `bbch_scope`. |
| **relationships** | Принадлежит `RapeseedTechCard`. Содержит `Operation`. Может иметь `ControlPoint`. |
| **lifecycle_position** | Часть жизненного цикла техкарты. |
| **mandatory** | ДА |

### 2.5. Операция (Operation)

| Атрибут | Значение |
|-----------|-------|
| **canonical_name** | `Operation` |
| **description** | Конкретное агрономическое действие (посев, внесение удобрений, опрыскивание). |
| **why_it_exists** | Атомарная единица полевых работ. Отслеживает, что должно быть сделано, с какими параметрами и качеством. |
| **key_attributes** | `operation_id`, `stage_id`, `name`, `operation_type` (Enum), `mandatory` (bool), `parameters` (JSON). |
| **relationships** | Принадлежит `TechCardStage`. Может требовать `InputMaterial`. Логируется в `ExecutionRecord`. |
| **lifecycle_position** | Определяется при генерации техкарты; выполняется в течение сезона. |
| **mandatory** | ДА |

### 2.6. Точка контроля (ControlPoint)

| Атрибут | Значение |
|-----------|-------|
| **canonical_name** | `ControlPoint` |
| **description** | Шлюз принятия решений на определенной фазе BBCH, где система собирает наблюдения и оценивает состояние. |
| **why_it_exists** | Реализует концепцию "шлюзов решений" — система не может слепо следовать плану без подтверждения реальности. |
| **key_attributes** | `control_point_id`, `stage_id`, `bbch_scope`, `required_observations` (list), `acceptance_ranges` (JSON). |
| **relationships** | Принадлежит `TechCardStage`. Может генерировать `Deviation` или `Recommendation`. |
| **lifecycle_position** | Определяется при генерации; оценивается в сезоне. |
| **mandatory** | Опционально для каждого этапа (но большинство этапов имеют его). |

## 3. Разграничение концепций (Concept Boundaries)

| Концепция A | Концепция Б | Описание различия |
|-----------|-----------|-------------|
| **Этап** (Stage) | **Операция** | Этап — логический блок с целью и окном времени. Операция — атомарное действие. Этап содержит 1-N операций. |
| **Правило** | **Рекомендация** | Правило — закодированная доменная логика. Рекомендация — контекстный совет, сгенерированный на основе правил и данных. |
| **ControlPoint** | **MonitoringSignal** | Точка контроля — запланированный шлюз в техкарте. Сигнал мониторинга — асинхронное событие из внешних систем (спутник, метео). |
| **Статичный атрибут** | **Наблюдение** | Статичный ( SoilProfile) меняется редко. Наблюдение (SeasonalObservation) — состояние культуры "здесь и сейчас". |
