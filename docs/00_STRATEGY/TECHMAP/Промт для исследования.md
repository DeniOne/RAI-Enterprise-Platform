РОЛЬ
Ты — Principal Agro-Architect + Chief Agronomist + Product/Legal Designer для RAI.
Твоя задача: спроектировать «Техкарту как стержень бизнеса» так, чтобы она одновременно была:
1) проектом урожая, 2) цифровым контрактом, 3) операционной моделью, 4) юридически значимым документом,
5) зафиксированной гипотезой достижения урожайности.
Техкарта уникальна для каждого урожая/поля/сезона/юрлица и не повторяется.

КОНТЕКСТ
- Культура: приоритет №1 — рапс (озимый и яровой), но архитектура должна масштабироваться на другие культуры.
- География консультирования: от Калининграда до Красноярска, от Кавказа (Краснодарский край) до Сибири.
- Контекст первичен: юрлицо/договорные условия → хозяйство → поле → предшественник → анализ почвы → семена → техника/люди → логистика → риски → ограничения.
- Методология централизована: есть единый «шаблон структуры» + правила расчёта; техкарта — вычисленный результат методологии.
- Процесс: ИИ формирует список недостающих данных для агронома → получает данные → строит техкарту с прогнозами/симуляциями → наш агроном утверждает → согласование с руководством хозяйства → техкарта уходит в исполнение и контроль.

ЦЕЛЕВОЙ ВЫХОД (WHAT TO DELIVER)
Сформируй полный результат в 8 блоках:

Блок A — Каноническая структура Техкарты (Table of Contents)
- Дай оглавление уровня 1–3 (разделы, подпункты).
- Укажи, какие разделы обязательны, какие опциональны, какие зависят от региона/типа рапса.
- Для каждого раздела: цель, кто заполняет, источник данных, как используется в исполнении.

Блок B — Модель данных (Data Schema)
- Опиши сущности и связи: LegalEntity, Farm, Field, Season, CropPlan, TechMap, Operation, Input, Application, Machinery, Labor, Risk, KPI, Observation, Evidence, Approval, ChangeOrder.
- Дай JSON-подобную схему (поля, типы, обязательность, диапазоны, единицы измерения).
- Отдельно: словари/enum’ы (фазы развития, виды операций, типы материалов, методы внесения, уровни доказательств).
- Отдельно: провенанс (provenance) и confidence для каждого критического поля (кто дал, когда, источник, уровень доверия).

Блок C — Методология расчёта и «урожай как гипотеза»
- Опиши, как техкарта фиксирует гипотезу: целевая урожайность, ожидаемый класс качества, допущения, ограничения, план питания/защиты/агротехники.
- Дай вычисляемые элементы: нормы, сроки, окна операций, зависимости, критические пороги (влага, температура, фенофаза, N/S/B, сорняки/вредители/болезни).
- Опиши «двухконтурность»: базовый план + адаптивные правила (триггеры: погода, NDVI, осмотры, фенология).
- Опиши «что считается ошибкой»: несовместимые смеси, превышение норм, пропуск критической операции, нарушение окна по фазе.

Блок D — Юридическая оболочка (Contract + Evidence)
- Опиши, что делает документ юридически значимым: версия, подписи, дата утверждения, стороны, ответственность, порядок изменений (Change Orders).
- Опиши структуру “подписываемой части” и “операционной части” (что фиксируем и подписываем; что может меняться по протоколу).
- Опиши требования к доказательствам исполнения: акты, фото/геометки, треки техники, накладные, лабораторные анализы, погодные данные.
- Опиши аудитный след: кто изменил, почему, на основании какого наблюдения.

Блок E — Операционная модель исполнения (Ops)
- Разверни техкарту в календарь/граф работ: задачи, зависимости, ресурсы, SLA, допуски по срокам.
- Для каждой операции: входные данные, pre-checks, допуски, контроль качества, post-checks, критерии завершения.
- Опиши протокол “ИИ → человек”: какие решения автоматизируются, какие только с подтверждением агронома/директора.

Блок F — Регионализация (Калининград–Кавказ–Сибирь)
- Сформируй механизм региональных профилей: климатические зоны/сумма температур/окна сева/риски перезимовки/засуха/болезни.
- НЕ пиши справочник конкретных дат по регионам “с потолка”.
- Дай архитектуру: где хранится региональный профиль, какие параметры влияют на расчёт, как обновляется.
- Дай пример 3 профилей: “морской/влажный”, “степной/засушливый”, “континентальный/холодный” — с параметрами (без конкретных «истинных» дат).

Блок G — Экономика, маржинальность, и контроль бюджета
- Опиши структуру бюджета по техкарте: семена, удобрения, СЗР, топливо, работы, аренда, логистика, анализы.
- Дай KPI: план/факт, отклонения, себестоимость/га, себестоимость/т, маржа, риск-коррекция.
- Опиши правила: что считается перерасходом, что требует согласования, как оформляется change order.

Блок H — Мини-пример (на рапс)
- Дай короткий демонстрационный пример (не “правильные нормы”, а шаблон заполнения):
  1) входной контекст (поле/почва/предшественник/семена/техника/ограничения),
  2) список запросов данных к агроному (в телеграм-стиле, чеклист),
  3) фрагмент техкарты: 8–12 операций с полями и зависимостями.

КРИТИЧЕСКИЕ ОГРАНИЧЕНИЯ (HARD RULES)
1) Не выдумывай агрономические нормы/сроки как “истину”. Вместо этого:
   - описывай структуру, параметры, диапазоны, правила вывода,
   - помечай места, где требуются региональные/сортовые/почвенные коэффициенты.
2) Всегда указывай единицы измерения и допустимые диапазоны.
3) Результат должен быть “инженерным”: таблицы/схемы/структуры/контракты, а не эссе.
4) Дай “Definition of Done” для структуры техкарты: что значит “структура готова к внедрению”.

ФОРМАТ ВЫВОДА
- Используй заголовки A–H.
- Внутри — списки, таблицы, JSON-схемы, чеклисты.
- Где уместно — добавь матрицы: (раздел → владелец данных → источник → контроль → доказательство).

НАЧИНАЙ С ТРИАЖА
Сначала сделай “карту неизвестного”: какие входные данные обязательны, какие желательны, какие можно оценивать/импутировать.
Затем выдавай блоки A–H.
