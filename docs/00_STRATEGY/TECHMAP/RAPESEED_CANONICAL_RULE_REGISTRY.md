# КАНОНИЧЕСКИЙ РЕЕСТР ПРАВИЛ ПО РАПСУ
## Централизованный репозиторий доменных правил для системы технологических карт рапса

**Версия:** 1.0.0  
**Домен:** Рапс (Brassica napus L.) — Озимый и яровой  
**Исходные модули:** M1 (Культура и условия), M2 (Идеальная технология озимого), M3 (Идеальная технология ярового), M4 (Международная практика), M5 (Каноническая техкарта и ядро)  
**Сгенерировано:** 2026-04-01  
**Статус:** Канонический — авторитарный источник для имплементации платформы RAI_EP

---

## 1. Назначение документа

Этот реестр является **единым источником истины** для всех контролируемых продуктовых правил в подсистеме техкарт рапса. Он служит официальным справочником для:

- Реализации движка правил на бэкенде
- Логики генерации технологических карт
- Валидации допуска полей и планирования посевов
- Внутрисезонной адаптации и контроля за отклонениями
- Интерпретации сигналов мониторинга
- Управления ручными переопределениями агронома
- Ограничений валидации UI-форм
- Контроля API-контрактов

Каждое правило отслеживается до исходных исследований и классифицировано по слою, типу, достоверности и политике переопределения.

## 2. Модель системы правил

### 2.1. Таксономия правил

| Тип правила (Rule Type) | Код | Описание | Автоматизация |
|-----------|------|-------------|------------|
| **Жесткий блокиратор** | `HARD_BLOCKER` | Абсолютный запрет. Система НЕ ДОЛЖНА продолжать выполнение. | Полностью автоматизировано |
| **Жесткое требование** | `HARD_REQUIREMENT` | Обязательное действие или условие. Не подлежит обсуждению. | Полностью автоматизировано |
| **Строгая рекомендация** | `STRONG_RECOMMENDATION` | Следует соблюдать; отклонение требует обоснования. | Полуавтоматизировано |
| **Условная адаптация** | `CONDITIONAL_ADAPTATION` | Логика ветвления на основе условий поля или сезона. | Автоматическое ветвление |
| **Сезонный триггер** | `SEASONAL_OVERRIDE_TRIGGER` | Динамический сигнал, модифицирующий план выполнения. | По событиям |
| **Сигнал мониторинга** | `MONITORING_SIGNAL` | Наблюдение, которое МОЖЕТ инициировать действие. | Генерация алертов |
| **Требуется ревью** | `HUMAN_REVIEW_REQUIRED` | Система не может принять решение автономно; нужен ввод агронома. | Ручной шлюз |

| Слой | Код | Описание |

| Уровень | Описание |
|-------|-------------|

## 4. Канонические пороги (Threshold Registry)

- **pH:** Критический `< 5.5`, Оптимальный `6.0-7.0`
- **Плотность почвы:** Если `> 1.5 г/см³` — обязательное рыхление.
- **САТ (SAT):** Яровой: мин `1650°C`. Озимый: мин `2200°C`.
- **Температурный стресс:** При цветении `> 30°C` — риск значительной потери урожая.
- **Влажность семян:** При уборке `9-12%`, для хранения `≤8%`.

## 5. Конфигурируемость правил

**Изменяемые параметры:** Пороги ЭПВ вредителей, дозировки NPK, окна посева в зависимости от зоны.
**Хардкод (Инварианты):** pH блокировка, севооборот, обязательность серы и бора, запрет инсектицидов в цветение.
