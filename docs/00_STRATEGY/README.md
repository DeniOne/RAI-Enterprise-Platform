# 00_STRATEGY

Активный слой стратегии, бизнес-логики и project intent.

Здесь живут:
- `BUSINESS` — бизнес-архитектура и стратегические основания
- `CONSULTING` — консалтинговая модель и IA
- `FRONT_OFFICE` — продуктовая логика front-office контура
- `STAGE 2` — agent platform, swarm governance и AI master-plan
- vision / roadmap / success metrics / переходные планы

Правило доверия:
- это активные документы, их нужно читать при проектировании, планировании и agent reasoning
- это не автоматическая runtime truth
- когда тезис из стратегии используется как утверждение о текущем поведении системы, его нужно перепроверять по `code/tests/gates` или по claim-managed operational docs
