# RBAC Matrix: Screens × Roles
RAI Enterprise Platform

## 0. Базовый принцип (обновлённый канон)

Пока в системе нет отдельных профильных руководителей,
мы используем 4 уровня доступа, соответствующие реальной ответственности:

*   **Администратор** — управляет системой
*   **Владелец / Генеральный директор** — управляет бизнесом
*   **Менеджер** — управляет хозяйствами и контрагентами
*   **Агроном (штатный)** — разрабатывает и сопровождает техкарты

👉 **По мере роста компании модель расширяется, а не ломается.**

## 1. Роли (текущая версия)

### 1.1 Администратор
*   **Назначение:** Техническое и системное управление платформой.
*   **Доступ:**
    *   полный доступ ко всем модулям
    *   системные настройки
    *   пользователи и роли
    *   интеграции
    *   аудит

### 1.2 Владелец / Генеральный директор
*   **Назначение:** Управление бизнесом и результатом.
*   **Доступ:**
    *   все функции системы
    *   все данные
    *   вся аналитика
    *   *без доступа к системным настройкам*

### 1.3 Менеджер
*   **Назначение:** Ведение хозяйств, контрагентов и контроль исполнения.
*   **Доступ:**
    *   CRM (хозяйства, контрагенты, поля)
    *   планы урожая
    *   контроль исполнения техкарт
    *   задачи, события, алерты
    *   просмотр результата по своим хозяйствам

### 1.4 Агроном (штатный)
*   **Назначение:** Консалтинг и разработка агрономических решений.
*   **Доступ:**
    *   разработка техкарт
    *   сопровождение исполнения техкарт
    *   контекст хозяйств, *только связанный с техкартами*
    *   фиксация отклонений и рекомендации

## 2. Обозначения в таблицах доступа

| Обозначение | Значение |
| :--- | :--- |
| **R** | Просмотр (Read) |
| **C** | Создание (Create) |
| **U** | Изменение (Update) |
| **D** | Удаление (Delete) |
| **CRUD** | Полный доступ: создание, просмотр, изменение, удаление |
| **–** | Нет доступа |

## 3. Управление Урожаем — RBAC

### 3.1 Обзор (Dashboard)
| Роль | Доступ |
| :--- | :--- |
| Администратор | Полный |
| Владелец / CEO | Полный |
| Менеджер | Просмотр своих хозяйств |
| Агроном | Просмотр своих техкарт |

### 3.2 Хозяйства и Контрагенты (CRM)
| Экран | Админ | CEO | Менеджер | Агроном |
| :--- | :--- | :--- | :--- | :--- |
| Реестр хозяйств | CRUD | R | R | R (ограничено) |
| Карточка хозяйства | CRUD | R | CRUD | R (контекст) |
| Контрагенты | CRUD | R | CRUD | R (контекст) |
| Поля / объекты | CRUD | R | CRUD | R |
| Импорт / ввод данных | CRUD | – | CRUD | – |

⚠️ *Агроном не управляет хозяйствами — только использует контекст.*

## 11. Каноническая формула RBAC

1.  **Администратор** управляет системой.
2.  **CEO** управляет бизнесом.
3.  **Менеджер** управляет хозяйством.
4.  **Агроном** управляет техкартой.

## 12. Вектор развития (зафиксировано)

При появлении:
*   директора по экономике
*   директора по GR
*   директора по производству

👉 **доступ выделяется из CEO,**
👉 **текущая модель остается валидной.**
