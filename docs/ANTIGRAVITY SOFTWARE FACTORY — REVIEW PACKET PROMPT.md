# ANTIGRAVITY SOFTWARE FACTORY  
## REVIEW PACKET PROMPT (CANONICAL) — СОБРАТЬ ПАКЕТ ДЛЯ РЕВЬЮ

## 0. СТАТУС
Этот промт применяется **сразу после реализации задачи** в Antigravity.
Цель — подготовить проверяемый пакет для внешнего ревью (Cursor-агент).

Запрещено: commit/push и “закрытие задачи” без ревью.

---

## 1. ВХОДНОЕ УСЛОВИЕ
Задача должна быть оформлена промтом в `interagency/prompts/YYYY-MM-DD_<slug>.md`.

Если файла нет — **PROCESS HALT**.

---

## 2. СОБРАТЬ ПАКЕТ НА РЕВЬЮ (BLOCKING)

### 2.1 Отчёт (обязателен)
- Создай/обнови: `interagency/reports/YYYY-MM-DD_<slug>.md`
- Формат: `interagency/templates/REPORT_TEMPLATE.md`
- Заполни фактами:
  - что было целью,
  - что сделано,
  - изменённые файлы (список),
  - проверки/прогоны (команды + результат),
  - что сломалось/не получилось,
  - следующий шаг.

### 2.2 Индекс (обязателен)
- Обнови `interagency/INDEX.md`:
  - ссылка на промт,
  - ссылка на план (если есть),
  - ссылка на отчёт,
  - статус: `READY_FOR_REVIEW` (не `DONE`).

### 2.3 Технические артефакты (обязательны)
Подготовь и приложи в конец отчёта (текстом, без ссылок на внешние системы):
- `git status` (вывод)
- `git diff` (полный или по ключевым файлам; если огромный — приложи список файлов + ключевые фрагменты)
- логи прогонов из тест-плана промта (тесты/линт/сборка)
- если в DoD есть manual check — строка “Manual check: PASS/FAIL + что проверено”

---

## 3. ЖЁСТКИЕ ЗАПРЕТЫ
- Нельзя делать `git commit`.
- Нельзя делать `git push` (в т.ч. без force).
- Нельзя ставить статус “DONE” в `interagency/INDEX.md`.
- Нельзя обновлять главные чеклисты и `memory-bank` на “готово” до ревью.

---

## 4. ФИНАЛ ВЫХОДА
Напиши короткое сообщение (3–8 строк):
- `READY_FOR_REVIEW`
- где лежит отчёт (`interagency/reports/...`)
- какие проверки прогнаны
- что осталось сомнительным/непроверенным
