Consulting Information Architecture (IA)
RAI Enterprise Platform

1. Назначение документа

Этот документ описывает экраны, рабочие места и навигационную логику консалтингового ядра RAI_EP.

IA строится не вокруг данных, а вокруг работы консультанта:

постановка задачи,

проектирование эталона,

контроль исполнения,

фиксация решений,

расчёт результата.

2. Канонический пользователь
Primary User

Консультант RAI

принимает ответственность за результат

проектирует План урожая

создаёт и ведёт Техкарту

принимает решения при отклонениях

Secondary Users

Руководитель практики (обзор, масштабирование)

Клиент (ограниченный read-only доступ)

Execution Agents (исполнители, ИИ, подрядчики)

11. Ключевой принцип IA

Любой экран либо помогает спроектировать эталон,
либо удержать факт в его границах,
либо честно посчитать результат.
