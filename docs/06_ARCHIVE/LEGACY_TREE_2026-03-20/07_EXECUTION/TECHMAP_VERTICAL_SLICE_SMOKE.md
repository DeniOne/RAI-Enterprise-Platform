# TechMap Vertical Slice Smoke

## Цель

Проверить рабочий путь `контрагент -> поле -> сезон -> план -> техкарта -> execution`
внутри UI без ручных REST-вызовов.

Эффект проверки:
- видно, как формируется техкарта из бизнес-контекста;
- видно, как выглядит состав карты: стадии, операции, ресурсы;
- видно, как карта попадает в execution-контур и как оттуда отслеживать исполнение.

## Маршрут

1. Открыть `/consulting/techmaps/demo`.
2. Выполнить по порядку:
   - `Создать контрагента`
   - `Создать поле`
   - `Создать сезон`
   - `Создать план`
   - `Сгенерировать техкарту`
3. На этом же экране перевести сущности:
   - `План -> REVIEW`
   - `План -> APPROVED`
   - `TechMap -> REVIEW`
   - `TechMap -> APPROVED`
   - `TechMap -> ACTIVE`
4. Открыть ссылку `Открыть техкарту`.
5. Открыть ссылку `Перейти в execution-хаб`.

## Что должно получиться

### После генерации TechMap

- создаётся не пустая запись, а карта со стадиями;
- у стадий есть операции;
- у операций есть ресурсы и плановые окна исполнения.

### На экране TechMap

- отображаются статус, версия, культура;
- видны счётчики стадий, операций и ресурсов;
- виден timeline по стадиям с операциями и нормами.

### После `TechMap -> ACTIVE`

- карта становится доступной для execution-модуля;
- появляется прямая ссылка на `/consulting/execution`.

### На экране execution

- операции из активной техкарты видны в общей ленте;
- их можно открыть как операционный объект исполнения.

## Технические примечания

- `POST /seasons` добавлен, поэтому сезон теперь создаётся из UI-клиента.
- `POST /tech-map/generate` теперь создаёт детерминированную непустую карту.
- `GET /consulting/execution/active` добавлен как alias к существующему execution endpoint, поэтому execution-хаб загружает данные без route mismatch.
