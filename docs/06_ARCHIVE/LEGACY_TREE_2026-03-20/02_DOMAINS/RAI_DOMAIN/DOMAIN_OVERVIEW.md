# RAI Domain Overview: Агрономическая логика 🌾

> **Статус:** Канон | **Версия:** 1.0 | **Слой:** Прикладной Домен (RAI)

---

## 1. Миссия Домена
**RAI Domain** — это "мозг" платформы, отвечающий за специфику сельского хозяйства. Его задача — превратить сухие данные (координаты, даты, нормы) в эффективные производственные процессы и высокий урожай.

Если **Business Core** отвечает на вопрос "Как работает система?", то **RAI Domain** отвечает на вопрос "Как вырастить рапс?".

---

## 2. Основные цели домена
1. **Цифровой двойник поля**: Точное представление земель компании в цифровом виде.
2. **Планируемость**: Описание технологических циклов (сезонов) для каждой культуры.
3. **Контроль исполнения**: Мониторинг всех полевых операций в реальном времени.
4. **Интеллектуальная поддержка**: Рекомендации по срокам посева, нормам высева и защите растений.

---

## 3. Архитектурная роль
RAI Domain является **потребителем** сервисов Ядра:
- Использует **Registry** для хранения иерархии полей.
- Использует **Task Engine** для запуска полевых работ.
- Использует **Identity** для назначения задач агрономам.

---

## 4. Контактная поверхность (Core Subsystems)
Домен расширяет базовые возможности Ядра:
- **Field** расширяет `Registry Node` через гео-координаты PostGIS.
- **Season** расширяет `Registry Context` через временные рамки и привязку к культуре.
- **Agro Operation** расширяет `Core Task` через специфические параметры (глубина заделки, норма СЗР).

---

## 5. Глоссарий (Базовый)
- **Поле (Field)**: Основной юнит производства. Имеет границы, площадь и историю.
- **Сезон (Season)**: Производственный цикл одной культуры на одном поле (напр., "Рапс 2024").
- **Культура (Crop)**: Растение со своими требованиями к севообороту и технологиям.
- **Техкарта (Technology Card)**: Идеальный план всех работ на сезон.
