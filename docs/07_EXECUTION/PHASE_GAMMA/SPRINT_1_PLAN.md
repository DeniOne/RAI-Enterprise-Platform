﻿# План Спринта 1 (Phase Gamma): Data & Infrastructure Readiness

**Срок:** 2 недели
**Цель:** Подготовка инфраструктуры, протоколов данных и наблюдаемости для подключения AI-солверов или финальных AI-моделей.

---

## Возможности и границы
- Глубокая AI-логика вынесена за рамки текущего планирования.
- Фокус инфраструктурный: контейнеры, память, логирование.
- Интеграция с Risk Engine в режиме read-only, только dry-run.
- Поля данных высокой достоверности (спутник, сенсоры, метеостанции) в статусе `TODO` и проектировании.

---

## Anti-Goals (чего точно не делаем)
- Обучение моделей или разработка inference-движков.
- Оценка точности/скорости готовых моделей.
- Бизнес-демонстрация или пользовательская приемка.
- Эксплуатация моделей экспертных или доменных.

---

## Объем работ

### 1. Инфраструктура (DevOps)
- K8s кластер/namespace для Gamma (`rai-gamma`).
- Настройка лимитов/квот (CPU/RAM/GPU) для изоляции.
- Secrets Management: перенос секретов в K8s Secrets.
- Подготовка базовых deployment шаблонов (для AI-моделей).

### 2. Наблюдаемость (Observability)
- Сквозной Trace ID на всех объектах (OpenTelemetry-compatible).
- Запись логов сессий и критических событий аналитики.
- Контур мониторинга первых этапов инжиниринга (latency/error-rate/throughput) для AI-задач.

### 3. Протоколы данных (Data Contracts)
- Унифицированные JSON-схемы (semver).
- Политика версионности: breaking change = новая major-версия.
- Хранение схем в Git как версионированных артефактов данных.

### 4. Хранилища (DB/Vector)
- Подключение pgvector (smoke test записи/чтения).
- Проектирование схем хранения векторных весов (для контекстной осведомленности).

### 5. Risk Engine (read-only)
- Реализация read-only endpoint для dry-run интеграции.
- Без влияния на систему, без вердикта.

---

## Приоритеты
- Инфраструктура: наблюдаемость, secrets, observability, data contracts.
- Готовность/доступность инструментов (для AI команд).
- Прохождение smoke tests (pgvector).
- Read-only интеграция Risk Engine (dry-run).

---

## Definition of Done
1. K8s среда подготовлена, namespace и квоты настроены.
2. Trace ID корректно проходит между звеньями.
3. JSON-схемы данных зафиксированы и лежат в Git.
4. pgvector smoke test пройден.
5. Risk Engine endpoint доступен в dry-run режиме, без записей.

---

## Риски и допущения
- Отработка на локальных и инфраструктурных K8s.
- Темп инжиниринга в условиях дефицита/недоступности.
- Темп разработки версионирования data contract схем.
