Твоя задача — реализовать пользовательский интерфейс для нового модуля "Коммерция" (Institutional Commerce Core) в платформе RAI Enterprise. Бэкенд (Prisma, сервисы) уже спроектирован и реализован. Твоя зона ответственности — фронтенд (сайдбар, роутинг, каркасы страниц).

---

## ШАГ 1: ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА UI DESIGN CANON (КРИТИЧЕСКИ ВАЖНО)
Любой написанный тобой код должен **СТРОГО** соответствовать дизайнерскому канону RAI Enterprise. Отклонения от канона являются архитектурным провалом.

1. **Базовый принцип:** Интерфейс должен быть лёгким, чистым, визуально "дышащим", без визуального шума. Акцент создаётся размером и пространством, а не жирностью.
2. **Цветовая модель (Light Theme):**
   - Строго светлая тема.
   - Запрещен чистый черный цвет (`#000` / `text-black`). Использовать смягченные оттенки (напр. `text-slate-900` или `text-black/80`).
   - Запрещены "тяжелые" серые фоны.
3. **Формы и контейнеры:**
   - Карточки и контейнеры: `bg-white`, `border-black/10`.
   - Скругления: строго `rounded-2xl` или `rounded-3xl` для крупных блоков.
4. **Типографика:**
   - Основной шрифт: `Geist`.
   - Заголовки, кнопки, метрики: `font-medium`.
   - Основной текст: `font-normal`.
   - **СТРОГО ЗАПРЕЩЕНО** использование `font-bold` и `font-semibold`.
   - Моно-шрифт (monospace) применять только для ID, логов и технических меток.
5. **Scorched Earth Policy:** При модификации старых компонентов (например, сайдбара или навигации) старый стиль полностью стирается и заменяется каноничным. Никакого смешивания legacy-стилей с новыми.

## ШАГ 2: CHECKLIST ИМПЛЕМЕНТАЦИИ (ПЛАН ДЕЙСТВИЙ)

Тебе необходимо выполнить следующие шаги:

### БЛОК А: Модификация Главного Сайдбара (Navigation)
- [ ] Встроить новую корневую кнопку **"КОММЕРЦИЯ"** в главный компонент сайдбара.
- [ ] **Точное позиционирование:** Кнопка "КОММЕРЦИЯ" должна располагаться **СТРОГО МЕЖДУ** разделом "УПРАВЛЕНИЕ УРОЖАЕМ" и разделом "СТРАТЕГИЯ" (т.е. точно над "Стратегией").
- [ ] Кнопка должна содержать релевантную легковесную иконку (например, `Briefcase`, `ShoppingBag` или `FileText` из пакета иконок, используемого в проекте, тонкая линия).

### БЛОК B: Вложенная маршрутизация (Sub-menu / Tabs)
- [ ] Под разделом "Коммерция" необходимо реализовать 4-компонентное подменю (в виде табов, дропдауна или элементов сайдбара — следуй текущей архитектуре навигации):
  1. **Договоры** (роут: `/commerce/contracts`) — Рамки и Роли
  2. **Исполнение договоров** (роут: `/commerce/fulfillment`) — Лента событий
  3. **Документы** (роут: `/commerce/invoices`) — Счета, Акты, ТТН, AR/AP
  4. **Оплаты** (роут: `/commerce/payments`) — Поступления и списания

### БЛОК C: Создание страниц-заглушек (Route Views)
- [ ] Создать 4 страницы (Pages/Views) по роутам, указанным выше.
- [ ] Каждая страница должна представлять собой чистый пустой Layout по канону: белый контейнер `rounded-3xl`, заголовок `font-medium` (не bold!), легкий boarder.
- [ ] Убедиться, что навигация между страницами Коммерции работает корректно и без перезагрузки (SPA-роутинг).

## ШАГ 3: КРИТЕРИИ ПРИЕМКИ (DEFINITION OF DONE)
1. Проект компилируется и собирается без ошибок линтера / TypeScript.
2. В главном меню появилась "КОММЕРЦИЯ" на правильном месте (строго перед "СТРАТЕГИЯ").
3. Навигация по 4 подразделам Коммерции работает и открывает пустые каноничные экраны.
4. В написанном тобой коде нет ни одного упоминания классов `font-bold` или `font-semibold`.
5. В интерфейсе экранов Коммерции нет чисто черного цвета, карточки имеют обязательные скругления `2xl`/`3xl`.
