# ARCHITECTURAL AXIOMS

## Статус документа
Данный документ фиксирует **НЕИЗМЕНЯЕМЫЕ архитектурные аксиомы** проекта.

- Не обсуждаются
- Не оптимизируются
- Не трактуются свободно

---

## 1. Аксиомы структуры

1. Система по определению модульная
2. Один модуль = одна зона ответственности
3. Модуль не знает о внутренностях других модулей
4. Связь между модулями только через зафиксированные интерфейсы

---

## 2. Аксиомы доступа

1. Нет прямого доступа к функциональности без Admission
2. Любой вход в систему проверяем
3. Любой доступ может быть отозван
4. Admission — блокирующий, а не рекомендательный механизм

---

## 3. Аксиомы данных

1. Данные имеют владельца
2. Источник данных всегда определён
3. Дублирование данных запрещено без причины
4. Неявные трансформации запрещены

---

## 4. Аксиомы процессов

1. Процесс важнее результата
2. Проверка предшествует действию
3. Отсутствие ошибки ≠ корректность
4. Повторяемость важнее скорости

---

## 5. Аксиомы ИИ-поведения

1. ИИ — исполнитель, не автор системы
2. ИИ не компенсирует пробелы мышлением
3. ИИ обязан останавливаться при неопределённости
4. ИИ подчиняется FOUNDATION, а не диалогу

---

## 6. Формула архитектуры

> **Детерминизм > Удобство > Скорость**
