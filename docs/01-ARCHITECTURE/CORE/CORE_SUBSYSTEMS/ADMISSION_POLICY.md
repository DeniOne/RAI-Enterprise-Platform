# ADMISSION × DECISIONS POLICY

## Статус документа
Данный документ формализует **обязательную связь Admission Gate и DECISIONS.log**.

- Является частью FOUNDATION
- Обязателен для всех Admission-процедур
- Нарушение считается критическим инцидентом

---

## 1. Базовый принцип

**Ни один Admission не считается валидным,  
если он не опирается на зафиксированное решение.**

DECISIONS.log — единственный допустимый источник
архитектурных и нормативных оснований Admission.

---

## 2. Когда DECISION REQUIRED

Проверка наличия решения в `DECISIONS.log` **обязательна**, если Admission касается:

- нового модуля
- нового типа доступа
- нового процесса
- нового API endpoint
- AI-агента или автоматизации
- изменения статуса доступа
- исключения из правил

Если затронут хотя бы один пункт — **Admission блокируется без Decision-ID**.

---

## 3. Admission Decision Check (ОБЯЗАТЕЛЬНЫЙ ШАГ)

Каждая Admission-процедура должна включать шаг:

DECISION CHECK


Состав проверки:

1. Указан ли `Decision-ID`
2. Существует ли запись в `DECISIONS.log`
3. Статус решения = `ACCEPTED`
4. Scope решения покрывает данный Admission
5. Admission не выходит за рамки решения

❌ Провал любого пункта → Admission = INVALID

---

## 4. Формат ссылки на решение

Admission **обязан явно ссылаться** на решение:

```text
Decision-ID: D-00X

Без указания Decision-ID Admission считается
архитектурно нелегитимным.

5. Admission WITHOUT DECISION (ЗАПРЕЩЕНО)

Запрещены Admission-процедуры:

«временные»

«пилотные»

«пока так»

«для MVP»

«мы потом зафиксируем»

Отсутствие решения не может быть компенсировано срочностью.

6. Исключения (ЕДИНСТВЕННЫЙ СЛУЧАЙ)

Единственное допустимое исключение:

Admission со статусом PENDING

С явной пометкой Decision Required

С блокировкой фактического доступа

Любая активность до принятия решения запрещена.

7. Ответственность ролей
USER

Инициирует принятие решения

TECHLEAD

Проверяет наличие Decision-ID

Блокирует Admission при отсутствии

Инициирует запись в DECISIONS.log

CODER

Не реализует Admission без Decision-ID

Обязан отказать в кодинге при нарушении

8. Ключевой принцип

Нет Decision-ID — нет Admission.
Нет Admission — нет доступа.


---

Admission без привязки к DECISIONS.log
считается архитектурным нарушением.
