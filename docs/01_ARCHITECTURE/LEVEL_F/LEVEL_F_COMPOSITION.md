### 4.2 Registry Cache Layer (Производительность)
- **Targeted Invalidation**: Система НЕ выполняет полный сброс кэша (`Full Purge`) при отзыве версии. Вместо этого инвалидируются только ключи, связанные с конкретной `formula_v`.
- **Soft-Expiry Pattern**: При истечении TTL (60s), система возвращает `stale` значение из кэша, инициируя фоновое обновление (`Background Refresh`) для предотвращения **Cache Stampede**.

### 4.3 SLA & Runtime Governance
- **Authority**: Изменение параметров SLA (лимиты QPS, таймауты) разрешено только роли `SRE_OPERATOR` с подтверждением через `Governance_Log`.
- **Throttling (Final)**:
  - `max_queries_per_second`: 100/sec.
  - `max_active_db_connections`: 10.
