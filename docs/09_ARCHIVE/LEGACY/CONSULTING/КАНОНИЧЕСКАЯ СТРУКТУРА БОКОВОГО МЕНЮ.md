0. Базовые принципы (обновлённые)

Мы управляем урожаем партнёров → мы управляем своим доходом.
Управление урожаем = управление результатом = управление экономикой.

Боковое меню отражает операционную модель консалтинга, а не IT-архитектуру.

CRM — не отдельный продукт, а входной контур управления хозяйствами и контрагентами.

Производство ≠ исполнение техкарт.
Производство — это физическое производство продукта «Грипил».

Исполнение техкарты — часть управления урожаем, с двумя ролевыми контурами.

2. Управление Урожаем — ЯДРО СИСТЕМЫ

Ассоциации:

управляем урожаем партнёров

управляем собственным финансовым урожаем

управляем результатом, а не процессом

3. Стратегия (бывший ОФС)

Только для руководителей.
Работает поверх Управления Урожаем.

Стратегия
 ├─ Стратегический обзор
 ├─ Портфель Планов Урожая
 ├─ Карта рисков
 ├─ Сценарное моделирование
 └─ Журнал стратегических решений

4. Экономика
Экономика
 ├─ Экономика урожая
 ├─ Агрегация эффекта
 ├─ Юнит-экономика
 ├─ Safety Net контроль
 └─ Прогноз экономики


Экономика считается из результата консалтинга, а не из бухгалтерии.

5. Финансы
Финансы
 ├─ Денежные потоки
 ├─ Начисления performance-вознаграждений
 ├─ Счета и расчёты
 ├─ Финансовые риски
 └─ Финансовая отчётность

6. GR
GR
 ├─ Регуляторный контекст
 ├─ Ограничения и требования
 ├─ Контракты
 ├─ Комплаенс
 └─ Юридически значимые решения


GR может:

накладывать ограничения на техкарты

инициировать отклонения

7. Производство (Грипил)

Не имеет отношения к техкартам и их исполнению.
Это отдельный физический контур.

Производство (Грипил)
 ├─ Сырьё и закупки
 ├─ Хранение
 ├─ Производство
 ├─ Контроль качества
 ├─ Логистика
 └─ Производственная аналитика

10. Ключевая формула Sidebar (обновлённая)
